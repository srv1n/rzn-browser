# EUI dashboard closeout friction

- `tusker automation plan` requires project knowledge paths that do not exist in this vault, so every packet reports a missing capsule before implementation can begin.
- `tusker finish --request-review` fails with `NOT_FOUND` until an attempt is manually started; the finish hint is useful, but the standard closeout loop should state that prerequisite.
- EUI verification rows prescribe direct Cargo commands while the repository policy requires `make rust ARGS=...`; task authoring should generate the policy-compliant form.
