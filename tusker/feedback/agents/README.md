# Agent Feedback

Use this folder for concise, actionable feedback to the Tusker builders.

Good feedback is about how Tusker helped or blocked agent work: finding the right task, understanding packets/context, using the CLI, handling state transitions, gates, proof/evidence, validation budgets, closeout, handoff, or bootstrap/PATH setup.

Do not use this folder for app/product feedback, feature ideas, implementation summaries, generic build failures, or UX notes from this repository unless the point is specifically how Tusker made that work harder, more repetitive, or unclear.

Create one Markdown file per meaningful work turn:

`YYYY-MM-DD-<agent-or-tool>-<short-topic>.md`

Keep each note under 10 lines:

- Context:
- Friction:
- Product idea:
- Impact:
- Related task or command:

Do not paste transcripts, logs, diffs, secrets, or raw tool output. Skip the note when there is no useful Tusker signal.
