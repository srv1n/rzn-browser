## EUI-T-0009 task-contract gap

The fleet window contract cannot be implemented in the extension-only scope. The fleet runner opens a normal browser session with an empty `SessionSpec`, so `browser.session_open`, `browser.execute_step`, and `browser.session_close` contain only a generated `session_id`; none identifies the run as fleet or carries its job/run id. `fleet_run_notice` is also notification-gated and has no session id or terminal keep-window decision, so it cannot safely associate a window with a run or drive cleanup when notifications are disabled. The task packet needs an explicit lifecycle envelope (origin + stable run/job id at session open, and terminal status + keep decision at session close) before the extension can preserve local-run behavior under concurrency.
