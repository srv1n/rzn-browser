# BRR-T-0001 · Auto-heal run-path bridge readiness before aborting

## Agent capsule

- Essence: Auto-heal run-path bridge readiness before aborting.
- Next action: define acceptance, do the smallest scoped change, and attach concise evidence.
- Read next: this note, then only the code/docs anchors named here.
- Avoid: raw logs, full transcripts, generated indexes, and attachments unless doing evidence forensics.

## Intent

Make the normal CLI run path recover from cold-start bridge unreadiness before
it asks the user to reload the extension. Today `native_runner.rs` calls
`runtime.ensure_ready` once, waits for the short readiness probe, and aborts
when the bridge is connected but unresponsive. This task makes pre-step failure
fall through to `runtime.heal` and retry readiness before any workflow side
effect is dispatched.

## Acceptance contract

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | `rzn-browser run` invokes `runtime.heal` once when initial readiness reports bridge connected but not ready/responsive. | Rust test or harness around native-run readiness flow proves ensure-ready false -> heal -> success proceeds. | `docs/features/bridge_reliability_hardening/README.md` task status. |
| 2 | If heal still fails, CLI reports the post-heal diagnostic rather than the initial generic readiness message. | Rust test covers failed heal output/error mapping. | Runtime docs if user-facing message changes materially. |
| 3 | Recovery happens only before the first workflow step is sent. | Code review/test confirms no side-effectful workflow step is replayed. | none |

## Symptom

- User sees `native_host_disconnected` / "loaded extension failed the readiness
  contract" even when the native-host fd is alive and the recoverable path is
  likely MV3 service-worker sleep or restart.

## Reproduction

1. Let Chrome/MV3 sit idle long enough for background service-worker eviction.
2. Run a CLI workflow through the supervisor path.
3. Observe `runtime.ensure_ready` return not ready after one 1.5s probe.

Expected:
- CLI attempts heal/reconnect before failing.

Observed:
- CLI aborts immediately and asks for extension reload.
