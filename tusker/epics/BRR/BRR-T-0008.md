# BRR-T-0008 · Add macOS and Windows bridge install doctor parity

## Agent capsule

- Essence: Add macOS and Windows bridge install doctor parity.
- Next action: define acceptance, do the smallest scoped change, and attach concise evidence.
- Read next: this note, then only the code/docs anchors named here.
- Avoid: raw logs, full transcripts, generated indexes, and attachments unless doing evidence forensics.

## Intent

Make bridge install and diagnostics equally trustworthy on macOS and Windows.
The runtime design above native messaging should be shared, while installer and
doctor checks must understand each platform's Chrome native-messaging
registration, process, path, and logging behavior.

## Acceptance contract

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Doctor validates native-messaging registration on macOS and Windows. | Unit/contract tests cover manifest path, allowed extension id, binary path, and missing/invalid registration cases. | Runtime docs include platform matrix. |
| 2 | Windows native-host stdio is binary-safe and app-base resolution matches CLI/supervisor/app. | Windows-focused test or CI job covers stdio mode/path resolution where available. | Install docs updated. |
| 3 | Status exposes supervisor/native-host executable path, build signature, app base, and extension build signature. | Supervisor/native-host tests cover stale binary/build mismatch reporting. | Bridge reliability docs mention stale-runtime fingerprinting. |
| 4 | Recovery handles platform process semantics. | macOS and Windows smoke plan proves stale native-host process can be terminated/recreated or reports exact manual action. | Doctor docs updated. |

## Scope

In:
- macOS NativeMessagingHosts manifest validation
- Windows registry-backed native messaging validation
- app-base/path/build-signature fingerprinting
- platform-specific stale native-host process diagnostics
- doctor output and remediation text

Out:
- replacing Chrome native messaging
- Playwright/WebDriver product-path substitution
- system-wide installer redesign beyond bridge requirements

## Deliverables

- doctor/install checks for both platforms
