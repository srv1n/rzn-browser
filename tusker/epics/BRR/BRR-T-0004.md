# BRR-T-0004 · Classify bridge readiness failures with typed diagnostics

## Agent capsule

- Essence: Classify bridge readiness failures with typed diagnostics.
- Next action: define acceptance, do the smallest scoped change, and attach concise evidence.
- Read next: this note, then only the code/docs anchors named here.
- Avoid: raw logs, full transcripts, generated indexes, and attachments unless doing evidence forensics.

## Intent

Replace the generic bridge readiness failure with typed diagnostics that map to
the likely fix. Sleeping SW, stale extension bundle, bridge down, and transport
timeout are different failures; collapsing them into "reload extension" wastes
debugging time.

## Acceptance contract

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Readiness output exposes distinct machine-readable causes such as SW asleep/unresponsive, stale bundle/capability mismatch, bridge down, and transport timeout. | Rust tests for readiness classification. | Runtime docs/status docs updated if schema changes. |
| 2 | CLI renders concise action text for each cause. | CLI/native-run test or snapshot covers message mapping. | Troubleshooting docs if user-facing copy changes. |
| 3 | Extension build signature/capability data is included when stale bundle is suspected. | Test fixture or smoke output shows expected vs loaded data where available. | `docs/features/bridge_reliability_hardening/README.md` task status. |

## Symptom

- The same user-facing message covers asleep service worker, stale fd, stale
  bundle, and true bridge-down states.

## Reproduction

1. Load an extension bundle without `content_keepalive_port` capability.
2. Separately force bridge ping timeout with fd still open.
3. Separately run with Chrome/extension unavailable.

Expected:
- Each scenario reports a distinct cause and next action.

Observed:
- Multiple cases ask for extension reload or report generic native-host
  disconnection.

## Scope

In:
- readiness/probe response structs
- CLI error rendering in the run path
- tests for classification and copy

Out:
- Changing capability policy; `BRR-T-0005` owns whether stale capability blocks
  all runs.
- Changing bridge teardown semantics; `BRR-T-0003` owns it.

## Deliverables

- Typed readiness error enum or equivalent structured cause.
