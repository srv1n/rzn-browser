# BRR-T-0007 · Add supervisor queue and profile bridge scheduler

## Intent

Move multi-producer browser work onto an explicit supervisor-owned queue and
scheduler. CLI, MCP, app, and cloud may all submit work and subscribe to
events, but only the supervisor should decide dispatch order, idempotency,
bridge lane ownership, and CDP/debugger leases.

## Acceptance contract

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Producers submit commands into a bounded supervisor queue with `producer_id`, `run_id`, and `command_id`. | Rust tests cover enqueue, terminal result replay, and duplicate `command_id` suppression before extension dispatch. | Bridge reliability docs mention supervisor fan-in. |
| 2 | Browser work is scheduled by profile/session/tab/side-effect class before reaching native host. | Tests show read-only work can coexist where safe and side-effectful work is serialized per session/tab. | Runtime canon updated if public status schema changes. |
| 3 | CDP/debugger actions use a per-tab lease with timeout cleanup. | Tests or harness prove a timed-out CDP action releases the lease and the next action can proceed. | CDP notes updated. |
| 4 | Consumers receive progress/results from supervisor event fanout, not from native-host broadcast. | Contract test covers two consumers subscribing to one run without duplicate extension dispatch. | Producer docs updated if tool output changes. |

## Scope

In:
- supervisor ingress queue
- session/profile bridge scheduler
- per-producer backpressure and cancellation
- per-tab CDP/debugger lease serialization
- event fanout from supervisor to local consumers

Out:
- making native host a producer bus
- direct producer-to-extension broadcast
- silent replay of side-effectful dispatched steps

## Deliverables

## Constraints

- One bridge authority per Chrome profile.
- Do not let native host broadcast to multiple producers.
- Never replay side-effectful commands after extension dispatch unless the
  command is explicitly idempotent.

## Escalate if

- A product requirement needs concurrent CDP/debugger ownership on the same tab
  target.
