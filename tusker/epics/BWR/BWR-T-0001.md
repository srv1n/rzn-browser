# BWR-T-0001 · Remove rzn-browser-worker from build, package, and release artifacts

## Agent capsule

- Essence: Remove rzn-browser-worker from build, package, and release artifacts.
- Next action: define acceptance, do the smallest scoped change, and attach concise evidence.
- Read next: this note, then only the code/docs anchors named here.
- Avoid: raw logs, full transcripts, generated indexes, and attachments unless doing evidence forensics.

## Intent

Stop producing or publishing `rzn-browser-worker` anywhere in the browser
runtime artifact pipeline. After this story, the worker binary is not built by
the browser plugin target, bundled into release artifacts, installed by runtime
installers, or listed as a shipped component.

## Acceptance contract

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Workspace/package scripts no longer build `rzn-browser-worker` as part of browser release/plugin targets. | `rg "rzn-browser-worker" Makefile scripts Cargo.toml scripts/plugins/config/rzn-browser.json` shows only removed-history/docs or intentional task notes. | Runtime/release docs stop listing the worker as shipped. |
| 2 | Browser plugin config ships `rzn-browser` and `rzn-native-host` only for runtime binaries. | JSON sanity check and plugin devkit tests cover the supervisor binary payload. | Plugin install docs describe supervisor binary discovery. |
| 3 | Bundle/install scripts do not require, copy, chmod, link, or print worker paths. | Dry-run or shellcheck-equivalent inspection of touched scripts plus `git diff --check`. | Bundle docs component list updated. |

## Scope

In:
- `Cargo.toml` workspace membership for the worker crate.
- `Makefile` plugin and demo targets that build/package worker.
- `scripts/plugins/config/rzn-browser.json`.
- `scripts/release/**`, `scripts/bundle/**`, and macOS package scripts.
- Release README/bundle component lists directly tied to shipped artifacts.

Out:
- Supervisor fallback code and CLI flags; BWR-T-0002 owns that.
- Native-run/native-host endpoint cleanup; BWR-T-0003 owns that.
- Deleting the worker source directory; BWR-T-0006 owns final source removal once consumers are gone.

## Deliverables

- Worker removed from release/plugin build commands and payload manifests.
- Worker removed from install/link/check paths.
- Tests/fixtures adjusted so devkit packaging still proves supervisor payloads.

## Constraints

- Do not remove generic plugin-worker support in rznapp.
- Do not remove `rzn-native-host`; it remains required browser transport.
- Keep artifacts installable from a clean checkout.

## Escalate if

- Any release path still requires a worker binary to build a browser artifact.
- Windows/Linux packaging depends on worker paths in a way that cannot be cleanly
  converted with the available source binary variables.
