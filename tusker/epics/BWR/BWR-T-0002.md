# BWR-T-0002 · Remove supervisor and CLI legacy worker fallback

## Agent capsule

- Essence: Remove supervisor and CLI legacy worker fallback.
- Next action: define acceptance, do the smallest scoped change, and attach concise evidence.
- Read next: this note, then only the code/docs anchors named here.
- Avoid: raw logs, full transcripts, generated indexes, and attachments unless doing evidence forensics.

## Intent

Remove the code paths that let supervisor and CLI browser commands fall back to
`rzn-browser-worker`. After this story, supervisor readiness and browser tool
execution are native-host-bridge or clear failure only.

## Acceptance contract

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Supervisor no longer exposes or evaluates `allow_legacy_worker_fallback`. | `rg "allow_legacy_worker_fallback|legacy_worker|call_browser_worker" crates/rzn_browser/src` returns no live fallback code. | Runtime docs say fallback was removed. |
| 2 | CLI/MCP browser surfaces no longer accept worker fallback flags/env vars. | CLI help/tests compile without fallback args; `rg "RZN_SUPERVISOR_ALLOW_LEGACY_WORKER_FALLBACK"` is empty outside history. | CLI docs updated. |
| 3 | Browser tools fail with supervisor/native-host bridge diagnostics only. | `cargo test -p rzn-browser supervisor` or targeted equivalent covers strict failure shape. | No separate docs beyond runtime/CLI docs. |

## Scope

In:
- `crates/rzn_browser/src/supervisor.rs`
- `crates/rzn_browser/src/main.rs`
- `crates/rzn_browser/src/mcp_browser.rs`
- rznapp supervisor call plumbing that exposes fallback to browser commands.

Out:
- Native-run worker attach/spawn helpers; BWR-T-0003 owns them.
- Release artifact packaging; BWR-T-0001 owns it.

## Deliverables

- Fallback config field/flag/env removed.
- Worker client call path deleted.
- Readiness/status JSON no longer reports legacy worker fallback modes.
- rznapp browser supervisor APIs stop accepting or forwarding legacy fallback.
