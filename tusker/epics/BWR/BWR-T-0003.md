# BWR-T-0003 · Remove native-run and native-host broker endpoint compatibility

## Agent capsule

- Essence: Remove native-run and native-host broker endpoint compatibility.
- Next action: define acceptance, do the smallest scoped change, and attach concise evidence.
- Read next: this note, then only the code/docs anchors named here.
- Avoid: raw logs, full transcripts, generated indexes, and attachments unless doing evidence forensics.

## Intent

Remove browser-runtime use of `broker_endpoint_v1.json` from native-run and
native-host paths. After this story, native host connects to supervisor socket
discovery only, and native-run/heal no longer attaches to or kills worker
endpoints.

## Acceptance contract

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Native host does not read broker endpoint files or attach to legacy browser bridge sockets. | `rg "read_broker_endpoint|broker_endpoint_path|LegacyBrowserBridgeV1" crates/rzn_native_host/src` is empty. | Native-host bridge docs describe supervisor-only discovery. |
| 2 | Native-run/heal no longer spawns, attaches, resets, kills, or prunes browser worker endpoints. | `rg "browser_worker|RZN_BROWSER_WORKER|broker_endpoint" crates/rzn_browser/src/native_runner.rs` has no live endpoint logic. | Browser native CLI docs updated. |
| 3 | Runtime status/heal output no longer contains broker endpoint prune reports. | Targeted Rust tests updated/removed. | Runtime docs updated. |

## Scope

In:
- `crates/rzn_native_host/src/main.rs`
- `crates/rzn_browser/src/native_runner.rs`
- runtime heal/status paths that surface endpoint prune data.

Out:
- Deleting the endpoint crate itself; BWR-T-0004 owns final deletion.
- Non-browser planner endpoint behavior in `rzn_plan`; decide explicitly before
  editing.

## Deliverables

- Native host supervisor endpoint discovery only.
- Native-run supervisor path only.
- Dead worker endpoint cleanup helpers removed.
- Tests updated from worker endpoint fixtures to supervisor socket/token fixtures.
