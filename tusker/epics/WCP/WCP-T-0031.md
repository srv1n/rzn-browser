# WCP-T-0031 · Publish workflow authoring gate for downstream agents

## Intent

Give workflow-building teams and agents one unambiguous standard: normal
workflow filenames, manifest-shaped JSON bodies, typed inputs/outputs, strict
validation, and inspect output before handoff.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Update authoring docs to remove `ManifestV2`/sidecar language from the public path.
- Provide a short checklist for new workflows and migrations.
- Show concrete examples using `google search` and one mutating workflow.
- Document `workflow inspect` vs `workflow validate` vs `workflow validate-catalog`.
- Document what must be true before broad rollout.

Out:
- Long architectural history of why sidecars were rejected.
- Internal Rust type rename from `WorkflowManifestV2`; user-facing docs should not leak it.
