# WCP-T-0030 · Certify migrated workflow pack against manifest standard

## Intent

Turn the in-place workflow migration into a release-quality proof: every
production workflow file uses the manifest standard, validates strictly, exposes
inspectable inputs/outputs, and a representative workflow matrix runs.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Confirm all production workflow JSON files use `schema_version: "rzn.workflow_manifest"`.
- Confirm no production `*.manifest.json` or `*.manifest.v2.json` sidecars remain.
- Run strict validation for every production workflow.
- Run `workflow inspect` checks for every route.
- Run a representative live/fixture smoke matrix across read-only, download, and mutating workflows.
- Split broad live-site validation into WAT tasks if the matrix grows beyond this task.

Out:
- Fixing engine bugs discovered by smoke; create/link WCP/WAT tasks instead.
- Performing irreversible third-party writes in smoke unless explicitly approved.
