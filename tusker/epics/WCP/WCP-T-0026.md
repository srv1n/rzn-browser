# WCP-T-0026 · Make manifest steps the native runner source

## Intent

Close the main runtime gap: manifest-shaped workflow files should be the runner's
authoritative input, not merely converted into the old
`browser_automation.sequences[].steps[]` shape before execution.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Load `WorkflowManifestV2.steps[]` directly for manifest workflows.
- Use `manifest.params` as the sole input contract for manifest workflows.
- Keep legacy workflow support only as an importer/backcompat path, not the preferred runtime model.
- Preserve existing browser executor behavior while removing author-visible dual sources of truth.
- Add one minimal manifest-only workflow fixture that runs without `browser_automation`.

Out:
- Rewriting every browser action implementation.
- Removing legacy workflow parsing for archived or user-local old files unless explicitly approved.

## Acceptance

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Manifest `steps[]` is authoritative for manifest workflows | A manifest-only fixture runs with no `browser_automation` object | Authoring guide says `steps[]` is executable source |
| 2 | Manifest `params` is authoritative | Tests show required/optional inputs come from `params`, not regenerated legacy var lists | Inspect docs stay current |
| 3 | Output selection validates against and reads manifest step ids | Strict validation fails on missing `result.output_selector.step_id`; runtime selects the declared step | none |
| 4 | Legacy adapter is isolated and marked transitional | Code path names/docs make fallback status explicit | none |

## Execution plan

- Split manifest workflow loading from legacy workflow loading.
- Keep conversion only at the final executor boundary if absolutely necessary.
- Add fixture coverage before touching production workflows.

## Evidence

- 2026-05-12 review rework: required CLI array params were impossible to pass because CLI params entered manifest normalization as strings while `ParamKindV2::Array` required JSON arrays.
- Native runner now coerces manifest array params from JSON arrays, comma-separated strings, or single values before contract normalization; invalid JSON arrays fail before runtime with a field-specific error.
- Corrected migrated scalar params that had been misclassified as arrays: `etsy/listing_details_reviews.listing_url`, `youtube/playlist.playlist_id`, `youtube/search.filter`, and `youtube/channel.mode`.
- Added regression coverage: `native_runner::tests::manifest_run_loader_accepts_cli_array_params`.
- Manifest workflows now load as `LoadedWorkflow` with `RuntimeStep::Manifest`; manifest params normalize through the contract schema.
- Legacy step conversion is isolated at the executor boundary for browser transport compatibility.
- Output selector validation rejects unknown manifest step ids.
- `cargo test -p rzn-browser native_runner -- --nocapture`: passed, 26 tests.
- `cargo test -p rzn-browser`: passed locally, 8 lib tests + 102 bin tests.
