# WCP-T-0028 · Enforce declared side effects end to end

## Intent

Turn manifest side effects from advisory metadata into enforced runtime policy,
including CLI post-processing writes and downloads.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Default manifest side-effect enforcement to on for CLI/host runs.
- Thread declared side effects into every step execution path, not only the happy-path runner.
- Track or block CLI post-processing side effects such as `--output-file` and `--download-dir`.
- Add an explicit unsafe/dev opt-out if needed.
- Emit useful errors when a workflow performs an undeclared effect.

Out:
- Human confirmation UX for destructive actions beyond the CLI/host policy primitive.
- New side-effect taxonomy names; those belong in WCP-T-0027.

## Acceptance

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Enforcement is on by default for manifest workflows | Test shows undeclared effect blocks without caller injecting `side_effect_policy.enforce:true` | Policy docs updated |

| 3 | CLI output/download writes are policy-visible | Test or smoke shows missing `file_write`/`download` declaration blocks post-processing | CLI docs updated if flag added |
| 4 | Debug opt-out is explicit | `--unsafe-*` flag or equivalent is named and tested, if introduced | CLI docs updated |

## Evidence

- Supervisor policy blocks undeclared observed step side effects and emits `RunResultV2` failure envelopes with workflow identity.

- `workflows/x/x_open.json` and `workflows/bing/bing_images_download.json` declare the new network/download policy surface.
