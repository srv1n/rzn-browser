# WCP-T-0025 · Make strict catalog validation a real rollout gate

## Intent

Make `workflow validate-catalog --strict` the release gate for the new workflow
manifest authoring standard. It must validate the production manifest set once,
reject malformed manifests, and avoid false duplicate failures from repo,
user, builtin, or legacy catalog roots.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Fix current strict manifest failures, starting with `workflows/chatgpt/chatgpt_send.json`.
- Define deterministic catalog source precedence for repo/user/builtin/legacy roots.
- Ensure duplicates caused by lower-priority shadowed sources do not fail strict validation.
- Preserve hard failure for true same-priority duplicate capability routes.
- Keep fixtures and archived workflows out of production catalog validation.

Out:
- Full live-site workflow smoke; that belongs in WCP-T-0030 / WAT.
- Runtime executor rewrites; that belongs in WCP-T-0026.

## Escalate if

- Source precedence would hide two different user-authored workflows with the same route.
- Catalog count changes because workflows were deleted or moved, not because shadowing is fixed.
