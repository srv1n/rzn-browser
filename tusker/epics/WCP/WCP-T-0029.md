# WCP-T-0029 · Make RunResultV2 the only host-visible run envelope

## Intent

Make CLI, MCP, and host consumers see the same typed `RunResultV2` envelope
with meaningful workflow identity, status, output, artifacts, warnings, and
debug metadata.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Prefer supervisor-provided `run_result` when present.
- Ensure CLI stdout emits `RunResultV2`, not legacy `{success,result,data}` payloads.
- Keep `--output-file` and download extraction unwrapping `RunResultV2.output`.
- Remove implicit success heuristics that convert legacy failure-shaped objects into success.
- Ensure `workflow_id` is the running manifest id, not `rzn.local.<tool>`.

Out:
- Changing the full artifact storage backend.
- UX redesign for rendered CLI output.

## Acceptance

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | CLI stdout is typed | Representative run emits `version:"rzn.run_result.v2"` | CLI/contract docs updated |
| 2 | Supervisor and CLI agree on status | Failure smoke returns `status:"failed"` and error details, no success wrapper | none |
| 3 | `workflow_id` is meaningful | Tests assert manifest id for step-level and workflow-level results | none |
| 4 | Output-file behavior remains ergonomic | `--output-file` writes `output`, not entire envelope, unless explicitly requested | CLI docs updated if option added |
