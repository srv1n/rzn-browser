# WCP-T-0023 · Make manifest canonical workflow contract and inspection surface

## Agent capsule

- Essence: Make manifest canonical workflow contract and inspection surface.
- Next action: define acceptance, do the smallest scoped change, and attach concise evidence.
- Read next: this note, then only the code/docs anchors named here.
- Avoid: raw logs, full transcripts, generated indexes, and attachments unless doing evidence forensics.

## Intent

Make `rzn.workflow_manifest` the workflow authoring and inspection surface:
typed inputs, declared side effects, executable manifest steps, selected typed
outputs, and RunResultV2-friendly CLI/host behavior. Treat legacy workflow JSON
as migration input/fallback, not the forward public format.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Acceptance

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | CLI/host can inspect workflow inputs, optionality, types, side effects, and outputs | `rzn-browser workflow inspect google search --json` returns typed manifest contract | Authoring guide documents command |
| 2 | Direct strict validation uses manifest contract validator | `rzn-browser workflow validate google search --strict --json` and direct manifest path return ok | README validation table updated |
| 3 | Catalog strict validation excludes fixtures and validates production manifests | `validate-catalog --strict --json` returns 80 manifests / 80 capabilities / 0 errors | Migration policy documented |
| 4 | Runner can execute manifest `steps[]` and thread policy context | `cargo test -p rzn-browser` covers manifest step loading, runtime identity, side-effect policy | Authoring guide says `steps[]` is canonical |
| 5 | Runtime output honors selected contract output and emits/unwraps RunResultV2 | Native runner tests cover selector output; CLI output helper unwraps `RunResultV2.output` | Output contract section added |
| 6 | Existing workflow pack has manifest coverage under old workflow filenames | migration script converted 80 workflow JSON files in place | Guide documents migration command |

## Scope

In:
- Manifest inspection command.
- Manifest-first catalog and named workflow resolution.
- Manifest strict validation for direct validate and catalog validate.
- Runner bridge from manifest steps to executable engine steps.
- Side-effect taxonomy alignment across manifests, runner payloads, supervisor policy.
- RunResultV2 output unwrapping and selector-aware runtime output.
- Bulk generated manifests for current workflow pack.
- Authoring docs for workflow builders and agents.

Out:
- Removing legacy workflow JSON files from the repository. They remain as temporary source/fallback until live-site smoke coverage is broader.
- Full live-site smoke of all 80 workflows.
