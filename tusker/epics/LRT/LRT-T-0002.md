# LRT-T-0002 · Implement durable rzn-browser supervisor runtime

## Intent
Build the long-lived `rzn-browser supervisor` runtime that becomes the canonical local owner for browser sessions, health, cloud actor state, and local producer IPC.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Add supervisor process mode to `rzn-browser`.
- Define and implement `rzn.local.v1` local IPC handshake and framing.
- Provide runtime status, ensure-ready, browser session, and heal entrypoints.
- Preserve compatibility with current CLI/native worker paths during migration.

Out:
- Reason app migration.
- MCP adapter migration.
- Full cloud actor move.

## Acceptance

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Supervisor starts idempotently and wins a single-user lock. | Unit/integration test plus manual `rzn-browser supervisor` smoke. | Update `local_supervisor_runtime` if startup policy changes. |
| 2 | Local clients can handshake and call `runtime.status` and `runtime.ensure_ready`. | IPC integration test. | Update local contract table. |
| 3 | Supervisor reports extension/native-host absent as degraded state, not crash. | Test with no extension bridge connected. | Update troubleshooting notes if needed. |
| 4 | Legacy endpoint discovery remains available only as fallback. | Existing native runner tests still pass. | No new docs unless behavior changes. |
