# LRT-T-0005 · Move cloud browser actor ownership into supervisor

## Intent
Move cloud browser actor polling/dispatch/result reporting out of the Chrome-owned native host and into the durable supervisor.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Rehost cloud actor config, token refresh, lease loop, command spool, and result reporting in supervisor.
- Preserve existing cloud command envelope semantics.
- Ensure cloud jobs use the same browser session path as CLI/MCP/app.

Out:
- Backend API redesign.
- Extension-direct cloud mode.

## Acceptance

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Supervisor owns cloud actor lifecycle and reports `cloud.status`. | Local fake cloud dispatcher test or smoke. | Update cloud control plane doc if flow changes. |
| 2 | Native host no longer starts cloud actor loop. | Code review/test with native host only. | Supersession note if needed. |
| 3 | Cloud command dedupe happens before extension dispatch. | Test duplicate command id. | Document dedupe contract. |
