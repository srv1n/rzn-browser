# LRT-T-0003 · Convert native host into thin extension-to-supervisor bridge

## Intent
Refactor the Chrome-launched native host so it is only the extension transport adapter and no longer owns durable runtime or cloud state.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Add native-host mode that connects extension frames to supervisor IPC.
- Register native-host bridge health with supervisor.
- Reconnect/backoff cleanly across supervisor and extension restarts.
- Keep Chrome native messaging manifest compatibility.

Out:
- Removing native messaging.
- Moving browser execution out of the extension.

## Acceptance

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | Native host forwards extension commands/results through supervisor. | Integration test or local smoke through extension. | Update bridge notes if protocol changes. |
| 2 | Native host restart does not lose supervisor run state. | Restart native host during idle and active-session smoke. | Document any unrecoverable active-step limits. |
| 3 | Native host does not own cloud lease loop. | Code review verifies cloud actor moved or disabled in native-host mode. | Cross-link cloud migration task. |
