# LRT-T-0007 · Migrate Reason app browser automation away from plugin worker dependency

## Intent
Make Reason.app a client of the shared browser supervisor instead of a process that owns or spawns a separate browser worker for automation.

## Read this when

Read this for task scope, acceptance, evidence, or implementation history.


## Scope

In:
- Identify current app plugin-worker browser paths.
- Add supervisor client integration strategy for `rznapp`.
- Preserve app-level tool registry/MCP UX where useful by wrapping supervisor browser tools.
- Remove hidden dependency on app-open state for CLI/MCP/cloud.

Out:
- Rebuilding the Reason app UI.
- Removing app-owned non-browser MCP/plugin capabilities.

## Acceptance

| # | Outcome | Proof required | Docs impact |
|---|---|---|---|
| 1 | App browser automation calls supervisor or a supervisor-backed adapter. | rznapp smoke or implementation PR evidence. | Update desktop bridge doc. |
| 2 | App no longer starts a separate durable browser worker for browser tasks. | Code review of plugin worker path. | Document migration behavior. |
| 3 | Existing app dev helpers can still call browser tools. | `window.__AGENT__` or equivalent smoke. | Update agent helper docs if needed. |
