# MBR-T-0014 · MB-015 scope bridge restart disconnect and pending-call drain

## Intent

Scope native-host bridge restart, disconnect cleanup, pending-call drain, and failure counters to the bridge that actually failed. A single browser bridge failure must not remove unrelated connected bridges or drain their pending calls.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | A timeout or sender failure on one bridge leaves another registered bridge usable. | Focused supervisor test |
| A2 | Restart requests are sent only to the bridge whose ID and epoch match the failed request. | Focused supervisor test |
| A3 | Pending calls for bridge B remain pending and can complete when bridge A disconnects or is restarted. | Focused supervisor test |
| A4 | Disconnect logs/health identify the failed bridge and per-bridge counters include restart, stale-drop, mismatch-drop, and timeout counts. | Focused supervisor test |
| A5 | No non-shutdown failure path clears all native bridges. | Focused supervisor test plus code inspection |

## Non-goals

- Public CLI target selection. That starts in MB-016/MB-024.
- End-to-end multi-browser smoke coverage. That is MB-034.
