# MBR-T-0023 · MB-024 add browser target flags to browser related CLI commands

## Intent

Add centralized browser target flags to browser-related CLI commands and pass target selection to the supervisor through one reserved routing object.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Commands that run workflows or call browser supervisor methods accept relevant target flags. | Focused CLI test |
| A2 | `--bridge`/`--browser-instance`/`--browser` conflicts are reported instead of silently ignored. | Focused CLI validation test |
| A3 | `--browser-instance` selects an instance and `--browser` selects browser kind through supervisor routing. | Focused supervisor routing test |
| A4 | CLI passes target data through a consistent `browser_target` object. | Focused payload-generation test |
| A5 | Help text includes short Edge/Chromium examples in flag descriptions. | Source/test review via focused parse coverage |
| A6 | Tests verify flag parsing and target payload generation for representative commands. | Automated test evidence |

## Non-goals

- One-browser default compatibility. That is MB-025.
- Native-host doctor diagnostics. That is MB-026.
- End-to-end browser-process smoke tests. That is MB-034.
