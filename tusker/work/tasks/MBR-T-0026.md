# MBR-T-0026 · MB-027 add structured bridge status API for supervisor clients

## Intent

Expose a stable machine-readable `runtime.bridges` supervisor API for SDK/MCP clients to inspect bridge identity, metadata, health, routing labels, and sessions without scraping human CLI output.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | API returns an array of bridge records with stable keys. | Focused contract test |
| A2 | Records include identity/metadata keys: `bridge_id`, `browser_instance_id`, `caller_origin`, `extension_id`, `extension_target`, `connected_since_ms`, and `last_successful_ping_at_ms`. | Focused contract test |
| A3 | Records include health counters for stale responses, restarts, timeouts, and active request count. | Focused contract test |
| A4 | API returns active session IDs/counts per bridge/browser instance. | Focused contract test |
| A5 | API returns no supervisor token, raw environment variables, or unnecessary local filesystem secrets. | Focused contract test |
| A6 | Contract tests validate the JSON shape. | Automated test evidence |

## Non-goals

- Human CLI formatting changes beyond the existing `browser targets` output.
- End-to-end browser-process smoke tests. That is MB-034.
