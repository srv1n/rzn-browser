# MBR-T-0038 · Extend native-host doctor to validate extension bundles

## Intent

Make doctor diagnose both native-host registration and the extension bundle users load.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Doctor checks bundle directory, `manifest.json`, manifest key-derived dev ID, native messaging permission, icons, background worker, and `rzn-build.json` target metadata. | Unit test or focused CLI test |
| A2 | Doctor output includes the exact load-unpacked directory for the target browser. | Unit test or snapshot |
| A3 | Missing bundle assets produce structured warn/fail checks instead of opaque errors. | Unit test |

## Non-goals

- Store ID verification.
- Full supervisor-connect self-test.
