# SEC-T-0012 · Harden cloud transport: https-only, token in header, remove legacy local cloud_set_config

## Intent

The cloud control-plane transport tolerates plaintext, puts the bearer token in URLs, and keeps a legacy path that lets the extension persist attacker-controlled cloud config without authorization. Enforce https/wss, move the token to a header, remove the legacy local config path, and give all cloud HTTP calls a shared client with timeouts.

## Context (read before coding)

Audit anchors (2026-06-11, re-verify). Note: `cloud::maybe_spawn_cloud_actor` is currently never called (dead code), but the config-persistence path executes and the actor may be wired later — fix all of it now.

Rust (`crates/rzn_native_host/src/cloud.rs`, `main.rs`):
1. `cloud.rs:568-575` — `websocket_url_from_server()` maps any non-https server_url to `ws://` (plaintext), carrying the token.
2. `cloud.rs:1026-1035` — `build_actor_ws_url()` appends `actor_token` as a query param; tokens in URLs land in proxy/access logs.
3. `main.rs:1983-1992` + `cloud.rs:677-709` — under env flag `RZN_NATIVE_HOST_LEGACY_CLOUD`, an extension-supplied `cloud_set_config` message persists fully attacker-controlled `server_url`/`websocket_url`/`actor_token` to disk with no authorization.
4. `cloud.rs:1056-1064` — `resolve_cloud_actor_config_path()` honors `RZN_CLOUD_CONFIG_PATH` env with no validation.

Rust (`crates/rzn_browser/src/cloud.rs`):
5. `:882, 909, 952, 979, 999, 1102` — `reqwest::Client::new()` per call, no timeout: hangs + new pool per request.

Extension (`extension/src/background.ts`):
6. `:7220` / `:7022` — popup-supplied `serverUrl` accepted over plain `http://`; the redeemed `actor_token` (`:7104`) then transits cleartext. **Sequencing**: SEC-T-0008 refactors this region of background.ts — land this task after it, or coordinate the diff.

## Implementation plan

1. **https/wss enforcement (host + extension)**: reject non-`https` server URLs unless host is loopback (`127.0.0.1`/`::1`/`localhost`) — loopback http stays allowed for local dev. Apply in `websocket_url_from_server` (https→wss only), in `cloud_set_config` validation, and in the extension's `fetchCloudJson`/pairing path.
2. **Token in header**: send `actor_token` via `Authorization: Bearer ...` header on the WebSocket handshake (tungstenite/`tokio-tungstenite` accepts a custom request builder — confirm which ws crate is used and build the handshake request with the header) instead of the query param. If the server side cannot accept headers yet, put it behind a `Sec-WebSocket-Protocol` subprotocol value as fallback — but prefer the header and note the server contract change in the evidence.
3. **Remove the legacy branch**: delete the `RZN_NATIVE_HOST_LEGACY_CLOUD` local handling of `cloud_set_config`; cloud control flows supervisor-only (the supervisor path already exists at `main.rs` dispatch). Grep for the env var name repo-wide (docs/scripts) and clean references.
4. **Config path validation**: `RZN_CLOUD_CONFIG_PATH` must resolve under the app config dir; otherwise ignore with a warning (keep an explicit dev escape hatch only if a test needs it — check tests first: `grep -rn "RZN_CLOUD_CONFIG_PATH" crates/`).
5. **Shared client**: one `OnceLock<reqwest::Client>` in `rzn_browser/src/cloud.rs` with `.timeout(30s)` and `.connect_timeout(10s)`; replace the 6 per-call constructions.
6. **Tests**: URL validation table test (https ok, http loopback ok, http remote rejected, ws downgrade impossible); handshake-builder unit test asserting the token appears in headers and NOT in the URL; config-path validation test.
