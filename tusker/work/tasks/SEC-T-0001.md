# SEC-T-0001 · Authenticate content/page bridges with per-load token and step allowlist

## Intent

Close the highest-severity finding from the 2026-06-11 audit: any web page (or cross-origin iframe) can drive the extension's privileged action handlers because the page-facing bridges have no authentication. Add a per-load secret token plus a step-type allowlist so only the engine can invoke privileged actions, and stop broadcasting results to `'*'`.

## Context (read before coding)

Line anchors are from the 2026-06-11 audit of `extension/src` — re-verify each anchor with grep before editing; code may have drifted.

Four unauthenticated page-reachable surfaces exist today:

1. `contentScript.ts:6382` — `window.addEventListener('message', ...)` accepts `RZN_TEST_PING` / `RZN_TEST_DOM_SNAPSHOT` / `RZN_TEST_EXECUTE` based on message **shape only**. The comment at 6385 says `event.source` comparison across MV3 worlds is unreliable, so all source/origin checks were dropped. `RZN_TEST_EXECUTE` dispatches `data.step` into `enhancedActionHandlers`/`actionHandlers` (~60 handlers at 6429-6447) including `eval_main_world`, `eval_isolated_world`, `get_cookies`, `set_cookie`, `get_local_storage_item`, `same_origin_request`, `take_screenshot`, `download_images`, `navigate_to_url`.
2. `contentScript.ts:6485` — DOM bridge: a MutationObserver on `#__rzn_page_bridge` executes any appended node with `data-rzn-type='execute'`; the only check is `data-rzn-target !== 'page'` (6493). Page MAIN-world JS can append a forged node and reach the same dispatch (duplicate handler-resolution block at 6519-6547).
3. `pageBridge.ts:694-783` — exposes `window.__rznEvalMainWorld`, `__rznEvalIsolatedWorld`, `__rznExecuteStep`, `__rznCaptureUiBundle` as non-writable MAIN-world globals on **every page** (manifest injects `<all_urls>`, `all_frames: true`).
4. Replies leak: results are sent via `window.postMessage(..., '*')` at `contentScript.ts:6398, 6408, 6414, 6460, 6467` — every frame on the page receives cookies/DOM/eval results.

Two adjacent integrity holes, fix in the same pass:

5. `contentScript.ts:5354-5357` — `executeWithSharedRequestDedup` trusts any node's `data-rzn-resp` in `#__rzn_content_script_execution_cache`; a page that pre-seeds a node matching the engine's req-id poisons results returned to the native host.
6. `contentScript.ts:5247-5251` — `isActiveContentScriptInstance()` reads a `documentElement` attribute the page can overwrite (DoS / guard bypass).

Why a cross-origin iframe matters: a hostile iframe can `window.top.postMessage(...)` and the TOP frame's content script executes the step in the **parent origin** — reading the parent's cookies/localStorage and running credentialed same-origin fetches. This is a real escalation, not self-eval.

Legit traffic note: the engine drives the content script through `chrome.runtime.onMessage` (listener around `contentScript.ts:5408-6379`) which web pages cannot reach, and through the pageBridge request-node protocol. The window-message bridge exists for tests. So gating the page-facing surfaces must not break the engine path — confirm by running the existing e2e suite.

## Implementation plan

1. **Token mint**: on content-script init, request a session token from background (`chrome.runtime.sendMessage({cmd:'get_bridge_token'})`). Background generates it with `crypto.getRandomValues` (32 bytes, hex) once per tab+document load, stores it keyed by `sender.tab.id` + frameId, and never exposes it to MAIN world or the page.
2. **Window-message bridge**: require `data.bridgeToken === token` for `RZN_TEST_*` messages. Keep the token in a module-level closure; never write it to the DOM. Test pages get the token from the test harness via the background channel, not from the page.
3. **Step allowlist**: add an explicit `PAGE_BRIDGE_ALLOWED_STEPS` set for anything dispatched from a page-originated channel (window message or DOM bridge). Deny by default; explicitly exclude: `eval_*`, `execute_javascript`, `get_cookies`, `set_cookie`, `clear_cookies`, `get_local_storage_item`, `set_local_storage_item`, `same_origin_request`, `take_screenshot`, `download_images`, `download_image`, `navigate_to_url`. Engine-originated dispatch (chrome.runtime path) is unaffected.
4. **Replies**: replace every `postMessage(..., '*')` with `postMessage(..., event.origin)` and include the request's token check before replying.
5. **DOM bridge + execution cache**: tag nodes the content script itself creates with `data-rzn-owner=<token-derived nonce>`; on read (dedup cache at 5354 and bridge dispatch at 6485) ignore nodes without a matching owner tag.
6. **Instance guard**: move `isActiveContentScriptInstance` state from the page-writable DOM attribute into extension-side state (closure variable + chrome.runtime coordination).
7. **pageBridge globals**: wrap the `window.__rzn*` helper installation in a build-time flag (`import.meta.env`/define from `build.sh`) so production builds do not install them; the DOM-node protocol remains for engine use.
8. Search for other `postMessage` targets and shape-only listeners across `src/content/` and `src/pageBridge.ts`; apply the same gate.
