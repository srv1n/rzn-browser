# MBR-T-0039 · Preserve Windows shared-manifest origins across browser installs

## Intent

Keep the dev shared Windows native-host manifest safe when Chrome, Edge, and Chromium installs are run separately.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Windows shared-manifest install unions existing `allowed_origins` with requested origins. | Unit test |
| A2 | Reinstalling for one browser does not remove origins required by another browser. | Unit test |
| A3 | Non-Windows manifest behavior is unchanged. | Unit test or source review |

## Non-goals

- Browser-specific Windows manifest paths for published installers.
- Registry integration tests requiring Windows.

## Verification

| Covers | Check | Result | Notes |
|---|---|---|---|
| A1-A2 | `cargo test -p rzn_sdk --features native_host windows_shared_manifest_install_unions_allowed_origins_across_browsers` | pass | Windows shared manifest keeps origins across Chrome then Edge installs |
| A3 | `cargo check -p rzn-browser -p rzn-native-host` | pass | Non-Windows registration paths still compile |
