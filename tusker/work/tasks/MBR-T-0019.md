# MBR-T-0019 · MB-020 define composite tab reference schema

## Intent

Define a globally unambiguous tab reference format that combines browser instance identity with browser-local numeric tab ID, while keeping numeric tab IDs backward-compatible when target context is already unambiguous.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Schema/model includes browser instance ID and numeric browser-local tab ID. | Focused contract test |
| A2 | Parser rejects malformed refs, missing instance ID, non-numeric tab ID, negative tab ID, and unsupported schemes. | Focused contract/TS tests |
| A3 | Serializer round-trips parser output exactly. | Focused contract/TS tests |
| A4 | Rust contracts and TypeScript extension types expose `tab_ref`. | Focused tests plus code inspection |
| A5 | Supervisor enriches tab-bearing results with `tab_ref` when browser instance context is known without removing numeric tab IDs. | Focused supervisor test |
| A6 | Same numeric tab ID in two browser instances yields distinct refs. | Focused contract test |

## Non-goals

- Requiring `tab_ref` as command input. That is MB-022.
- Replacing all numeric `tab_id` output. That is MB-021.
