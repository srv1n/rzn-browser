# EUI-T-0006 · Dashboard Workflows tab with health and run-now

## Intent

The dashboard Workflows tab: one place to see every workflow available on this device (local files + server-cached), its health flag with the dominant failure fingerprint, and a "run now" form. This is where a degraded/broken workflow becomes visible to a human.

## Context

- Needs one new supervisor RPC (owned here): `workflows.list {} -> {workflows: [{workflow_id, name?, source: "local"|"server_cache", workflow_hash, version?, last_run_at?, health?}]}` — merge the local catalog (`workflow_catalog.rs`), the fleet workflow cache (`workflow_cache.rs`), and health from `workflow_health` (EUI-T-0002). Dedupe by workflow_id preferring the entry that actually runs.
- `runs.start {workflow_id, params?}` exists (EUI-T-0003).
- Existing `workflow_params.rs` may define param conventions — check before inventing a form model.

## Contract

- Supervisor side: `workflows.list` RPC arm as above (implementation may live in a small `workflows_rpc` module or alongside `supervisor_control.rs` — implementer's call, but keep it out of `supervisor.rs` bulk).
- `#workflows` list: name/id, source chip, short hash, last run (relative), success sparkline (last ≤20 runs from health), flag chip — healthy (green) / degraded (yellow) / broken (red). Broken/degraded rows show the dominant fingerprint as a plain sentence: "failed 4× at step 6 (selector_not_found) since Jul 9".
- Run-now: per-row button opens a params form — key/value rows plus a raw-JSON toggle (zod/JSON.parse validation with inline error); submit → `runs.start` → deep-link to `#runs/<run_id>`. While paused, the button is disabled with an explanatory tooltip.
- Stretch (non-blocking): "Report broken" affordance on flagged rows wiring to the existing `workflow_failure_report.rs` submit path via a small RPC — only if trivially reachable; otherwise leave a TODO note in the attempt.

## File ownership

- OWN: `extension/src/dashboard/workflows/**` (new), `workflows.list` RPC arm + merge implementation (supervisor side).
- SHARED (additive): `extension/src/ui/rpc.ts` schemas.
- OFF-LIMITS: `workflow_catalog.rs` / `workflow_cache.rs` internals (read-only use), other dashboard tabs.

## Constraints

- `workflows.list` must not hit the network (local catalog + cache + run store only).
- No new dependencies. No AI attribution anywhere.

## Dependencies

- EUI-T-0004 (scaffold), EUI-T-0002 (health), EUI-T-0003 (`runs.start`).
