# SEC-T-0010 · Close plugin devkit verify bypass for unlisted zip entries and fix parse_mode octal handling

## Intent

`rzn_plugin_devkit verify` reports success on plugin bundles that carry unsigned extra payloads, because it only checks ZIP entries listed in the manifest's sha256 map. Separately, `parse_mode` silently mangles permission bits for JSON-number input. Close both.

## Context (read before coding)

Audit anchors (2026-06-11, re-verify) — all in `crates/rzn_plugin_devkit/src/main.rs`:

1. `:806-870` — `verify` iterates manifest-listed entries and checks their hashes, but ZIP entries **not present** in the `sha256` map pass silently. Entries with unsafe names (`../escape`, absolute paths, backslash separators) are also not rejected. Result: a "verified" bundle can contain arbitrary unsigned files, and whatever later extracts the bundle may write outside the target dir.
2. `:670-685` — `parse_mode`: a JSON **number** `755` is parsed as decimal 755 = `0o1363`, while the **string** `"755"` is parsed as octal `0o755`. Same author intent, wildly different bits, no warning.
3. (Cross-ref: the private-key 0644 write at `:223` is fixed by SEC-T-0005 — don't double-fix, but rebase if it landed.)

## Implementation plan

1. **Strict entry set in `verify`**: build the allowed-entry set = manifest hash-map keys ∪ {`plugin.json`, `plugin.sig`} (confirm exact metadata filenames from the `pack` code path in the same file). Any ZIP entry outside the set ⇒ verification failure naming the entry. Directory entries (`foo/`) may pass only if their files are listed.
2. **Name safety**: reject any entry whose normalized name contains `..` components, starts with `/` or a drive letter, or contains `\`. Apply in both `verify` and any extraction path in this crate (`grep -n "extract\|unpack" src/main.rs`).
3. **parse_mode**: accept only octal strings (`"755"`, optionally `"0o755"`); a JSON number becomes a hard error with the hint `use a string like "755"`. Grep the repo's plugin manifests/fixtures for numeric `mode` values and migrate them in the same change.
4. **Tests**: craft fixture zips in a tempdir within tests — (a) valid bundle passes; (b) bundle + one unlisted extra file fails naming the file; (c) bundle with `../evil` entry fails; (d) parse_mode: `"755"` → 0o755, number 755 → error, `"abc"` → error.
