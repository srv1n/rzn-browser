# FLA-T-0004 · Workflow manifest cache by content hash with GC

## Intent

Implement the on-device workflow manifest cache: content-hash-addressed storage under the runtime dir, fetch-on-miss from the backend with hash verification, and garbage collection of stale versions. Jobs reference `(workflow_id, content_hash)`; the cache guarantees the device runs exactly the bytes the server dispatched.

## Context

- Cross-repo hash contract (backend FLT-T-0004): content hash = sha256 hex of the COMPACT `serde_json::to_vec` serialization of the parsed manifest JSON value. The device re-serializes the fetched `manifest` value compactly and verifies sha256 == requested hash before trusting or storing it.
- Fetch endpoint: `GET {server}/v1/fleet/workflows/:workflow_id/:hash` with device-token bearer → `rzn_contracts::fleet_v1::FleetWorkflowFetchResponseV1`.
- Runtime dir helpers live in `rzn_core::runtime_paths`; file writes should be atomic (write temp + rename) and use the repo's secure-file conventions where applicable (`rzn_core::secure_files`).

## Contract

New `crates/rzn_browser/src/workflow_cache.rs`:

```rust
pub struct WorkflowCache { root: PathBuf /* <runtime>/workflow_cache */ }

impl WorkflowCache {
    pub fn new(root: PathBuf) -> Self;
    /// Cache hit: parse and return. Miss: fetch via `fetcher`, verify hash, store atomically, return.
    /// Hash mismatch on fetch → error (never store); corrupt cached file → evict + refetch once.
    pub async fn get(&self, workflow_id: &str, content_hash: &str, fetcher: &dyn ManifestFetcher)
        -> Result<serde_json::Value, CacheError>;
    /// Keep the newest `keep_per_id` hashes per workflow_id (by mtime), delete the rest;
    /// also delete any version older than `max_age_days` regardless. Returns stats.
    pub fn gc(&self, keep_per_id: usize, max_age_days: u64) -> GcStats;
}

#[async_trait]
pub trait ManifestFetcher: Send + Sync {
    async fn fetch(&self, workflow_id: &str, content_hash: &str)
        -> Result<FleetWorkflowFetchResponseV1, FetchError>;
}

pub fn manifest_content_hash(value: &serde_json::Value) -> String; // the shared algorithm
```

Layout: `<root>/<workflow_id_sanitized>/<content_hash>.json` — sanitize workflow_id to `[a-z0-9_-]` (reject path separators outright; ids come from the server but defense-in-depth). Touch the file's mtime on every cache hit so GC keep-newest reflects usage. The real `HttpManifestFetcher` (reqwest + device token from fleet config) also lives in this file; FLA-T-0003's loop injects it and calls `gc(3, 30)` once per day.

Tests (mock fetcher, tempdir): miss→fetch→verify→store→hit-without-fetch; hash-mismatch fetch rejected and nothing stored; corrupt cached JSON evicted and refetched; GC keeps newest N per id and respects max age; sanitization rejects `../` ids.

## File ownership

- OWN: `crates/rzn_browser/src/workflow_cache.rs` (new), the `mod workflow_cache;` crate-root line.
- OFF-LIMITS: `supervisor_fleet.rs` (FLA-T-0003 wires the cache in), `workflow_runner/**`, everything else.

## Constraints

- No panics on filesystem errors — degrade to refetch or surface `CacheError`.
- Hash verification is mandatory on every fetch; a served manifest that does not hash to the requested value is a hard error (`CacheError::HashMismatch`) — this is the integrity boundary of the fleet design until manifest signing lands.
- No AI attribution anywhere.
