# MBR-T-0025 · MB-026 add native host doctor command

## Intent

Add a native-host doctor command that diagnoses manifest registration, allowed origins, host executable health, supervisor reachability, and connected bridge inventory for a requested browser/extension origin.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Doctor reports `pass`, `warn`, or `fail` for each check. | Focused doctor test |
| A2 | Doctor detects missing manifest/registry registration. | Focused doctor test |
| A3 | Doctor detects missing requested extension origin in `allowed_origins`. | Focused doctor test |
| A4 | Doctor detects missing or non-executable host path on Unix-like systems. | Focused doctor test |
| A5 | Doctor reports supervisor socket/token reachability without printing token values. | Focused doctor test |
| A6 | Doctor reports whether a bridge from requested extension/browser is connected and supports JSON/human output. | Focused doctor test |

## Non-goals

- Launching browsers for end-to-end smoke tests. That is MB-034.
- Rewriting native-host install/uninstall semantics.
