# MBR-T-0018 · MB-019 support safe same-instance reconnect for bound sessions

## Intent

Let bound browser sessions survive reconnects from the same browser instance while refusing automatic rebinds to a different or suspicious browser identity.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | A session bound to browser instance X can continue on a new bridge for X after the old bridge disconnects. | Focused supervisor test |
| A2 | A session bound to browser instance X does not route to connected browser instance Y while X is disconnected. | Focused supervisor test |
| A3 | Same browser instance ID with conflicting caller origin is refused and marked suspicious. | Focused supervisor test |
| A4 | Session status records show disconnect/reconnect activity. | Focused supervisor test |
| A5 | Commands during the disconnected window return `BROWSER_INSTANCE_NOT_CONNECTED`. | Focused supervisor test |

## Non-goals

- Public session inspection CLI. That can build on the status record later.
- End-to-end MV3 lifecycle smoke tests. That is MB-034.
