# MBR-T-0037 · Reject ambiguous untargeted multi-browser routing

## Intent

Prevent untargeted commands from silently choosing Chrome when more than one browser bridge is connected.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | With zero bridges, the existing no-bridge error remains clear. | Unit test |
| A2 | With one bridge, untargeted commands still use that bridge. | Unit test |
| A3 | With two or more bridges and no explicit/saved target, target resolution returns `AMBIGUOUS_BROWSER_TARGET`. | Unit test |
| A4 | Saved default behavior remains available and deterministic only when the saved target resolves. | Unit test |

## Non-goals

- Adding more browser families.
- Changing explicit `--browser`, `--browser-instance`, or `--bridge` semantics.

## Verification

| Covers | Check | Result | Notes |
|---|---|---|---|
| A1-A3 | `cargo test -p rzn-browser bridge_target` | pass | 3 focused bridge target tests passed, including multi-bridge default ambiguity |
| A4 | `cargo test -p rzn-browser preferred_browser_target` | pass | 3 preferred/saved default routing tests passed |
