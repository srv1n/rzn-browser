# MBR-T-0020 · MB-021 return composite tab refs in tab session workflow results

## Intent

Return globally unambiguous tab references in tab/session/workflow results so CLI users can safely reuse a tab identity when multiple browser instances are connected.

Story dependency: MB-020 / MBR-T-0019 must exist in the working tree; it is already implemented and in review.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | `open_new_tab` results include numeric `tab_id`, `browser_instance_id`, and composite `tab_ref`. | Focused supervisor/extension test |
| A2 | Snapshot/session status style results include `tab_ref` when a current tab is known. | Focused supervisor/formatter test |
| A3 | Chrome and Edge results with the same numeric tab ID produce different `tab_ref` values. | Focused supervisor test |
| A4 | Existing consumers reading `tab_id` or `current_tab_id` continue to work in one-browser mode. | Existing broad tests plus focused assertion |
| A5 | Human-readable result formatting includes enough browser/tab context to distinguish browsers without dumping noisy JSON. | Focused formatter test |
| A6 | Representative workflow/browser result tests prove tab-ref presence. | Automated test evidence |

## Non-goals

- Accepting `tab_ref` as command input. That is MB-022.
- Adding public browser target inspection commands. That is MB-023.
- Removing numeric `tab_id` or `current_tab_id` compatibility fields.
- End-to-end multi-browser smoke tests. That is MB-034.
