# MBR-T-0021 · MB-022 accept tab_ref command input and reject ambiguous numeric tab ids

## Intent

Accept composite `tab_ref` input for tab-targeting browser commands and reject unsafe bare numeric tab IDs whenever multiple connected browser instances make that ID ambiguous.

Story dependency: MB-021 / MBR-T-0020 must exist in the working tree; it is already implemented and in review.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | `--tab-ref rzn://browser/<instance>/tab/123` resolves to browser instance plus numeric browser-local tab ID. | Focused CLI/supervisor test |
| A2 | `--tab 123 --browser edge` is accepted when Edge resolves to exactly one connected bridge. | Focused supervisor test |
| A3 | `--tab 123` with multiple connected browser instances returns `AMBIGUOUS_BROWSER_TARGET`. | Focused supervisor test |
| A4 | Malformed `--tab-ref` returns `INVALID_TAB_REF` with a short format example. | Focused parser/CLI test |
| A5 | The extension receives only the numeric tab ID after supervisor bridge resolution. | Focused supervisor dispatch test |
| A6 | Tests cover tab-ref routing and ambiguous numeric tab rejection. | Automated test evidence |

## Non-goals

- Public browser target inspection commands. That is MB-023.
- Replacing all numeric tab IDs; numeric IDs remain valid when scoped to one browser target.
- End-to-end multi-browser browser-process smoke tests. That is MB-034.
