# MBR-T-0022 · MB-023 add browser targets bridges inspection command

## Intent

Add an operator-facing command and supervisor method that lists connected browser bridges and browser instances so users can choose valid browser targets before routing commands.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | With zero bridges connected, the command returns an empty list and a clear status. | Focused supervisor/formatter test |
| A2 | With Chrome and Edge connected, the command lists two separate records. | Focused supervisor test |
| A3 | Each record includes identifiers usable with `--bridge` and `--browser-instance`. | Focused supervisor test |
| A4 | Human-readable output is compact and does not expose supervisor tokens or sensitive local paths. | Focused formatter test |
| A5 | JSON output includes full structured metadata for automation. | Focused supervisor/CLI test |
| A6 | Tests cover empty, one-bridge, and two-bridge outputs. | Automated test evidence |

## Non-goals

- Adding target flags to all browser-related CLI commands. That is MB-024.
- Native-host install doctor diagnostics. That is MB-026.
- End-to-end browser-process smoke tests. That is MB-034.
