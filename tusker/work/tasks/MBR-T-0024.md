# MBR-T-0024 · MB-025 preserve backward compatible one browser defaults

## Intent

Preserve the existing one-browser CLI and supervisor behavior while making only multi-bridge ambiguity require explicit target flags.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | With exactly one connected bridge, existing commands without target flags route successfully. | Focused compatibility test |
| A2 | With zero bridges, existing commands fail with the clear no-bridge error. | Focused compatibility test |
| A3 | With multiple bridges, existing commands fail with ambiguity instead of selecting a last connected bridge. | Focused compatibility test |
| A4 | Existing result fields such as `current_tab_id` remain present where previously present. | Focused compatibility test |
| A5 | Compatibility tests cover session open, snapshot, execute step, and close in one-bridge mode. | Automated test evidence |
| A6 | Release notes call out only the multi-bridge ambiguity behavior as a breaking-ish operational change. | Docs diff plus automated check |

## Non-goals

- Changing explicit target flag semantics. That is MB-024.
- Native-host doctor diagnostics. That is MB-026.
- End-to-end browser-process smoke tests. That is MB-034.
