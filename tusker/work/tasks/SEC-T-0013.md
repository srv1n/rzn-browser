# SEC-T-0013 · Repo hygiene and CI gates: untrack artifacts, clippy -D warnings, audit gate, workflow JSON validation

## Intent

Stop generated artifacts from living in git, and turn the advisory-only CI checks into real gates: clippy must fail on warnings, cargo-audit must gate on a baseline, and workflow JSON must be validated in CI (today ~40% of the 82 manifests have zero parse coverage).

## Context (read before coding)

Audit anchors (2026-06-11, re-verify):

1. `rzn-browser-code-2026-05-15-2.zip` (833KB) tracked at repo root — a stale source snapshot.
2. `extension/dist-chrome/` and `extension/dist-firefox/` are tracked: `.gitignore` line ~2 has `dist/` which does not match `dist-chrome`/`dist-firefox`. Built bundles drift from `extension/src`.
3. `tusker/Attachments/LRT-T-0010/dist-chrome` is tracked as a 0-byte file (broken artifact).
4. `.github/workflows/ci.yml:42` — clippy runs without `-D warnings`; `:149` — cargo-audit has `continue-on-error: true`. Neither can fail a build. Known existing advisories to triage: `ansi_term` (RUSTSEC-2021-0139) via rzn_plan.
5. No CI job validates workflow JSON. Rust parse tests (`*_workflows_parse_test.rs`) cover only google/hn/reddit/x/chatgpt/claude; amazon, etsy, g2, instagram, bing, youtube, finance, airbnb etc. are uncovered. `crates/rzn_browser/src/workflow_validator.rs:139` validates programmatically — there is no JSON Schema for `rzn.workflow_manifest`.
6. Doc nits to fix in passing: README workflow table lists Google "Shopping" but `workflows/google/google-search.json:198` says shopping is intentionally excluded; `.env.example:11,18` advertises 2025-era model ids/pricing.

## Implementation plan

1. **Untrack artifacts**: `git rm --cached rzn-browser-code-2026-05-15-2.zip` and delete it; `git rm --cached -r extension/dist-chrome extension/dist-firefox` (confirm with the human first ONLY if some install flow consumes the tracked dists — check `grep -rn "dist-chrome\|dist-firefox" scripts/ Makefile setup.sh install.sh` and the extension `build.sh`; if an installer expects them, document that instead of untracking and say so in evidence). Remove the 0-byte Attachments file. Update `.gitignore`: `extension/dist*/`, `*.zip` at root scope.
2. **Clippy gate**: ci.yml clippy step → `cargo clippy --workspace --all-targets -- -D warnings`. Run locally first; fix or `#[allow]`-with-comment any existing warnings so the gate lands green (audit ran clippy clean recently, expect few).
3. **Audit gate**: drop `continue-on-error`; add an `audit.toml` ignoring currently-unfixable advisories with a comment + date each (e.g. RUSTSEC-2021-0139 until ansi_term is replaced in SEC-T-0014/backlog). New advisories then fail CI.
4. **Workflow validation in CI**: prefer a glob-driven Rust test in rzn_browser that walks `workflows/**/*.json` (excluding `fixtures/` envelope files with different schema_version — handle `rzn.workflow_manifest` v1 + v2) and runs `workflow_validator` on each; this covers all 82+ manifests forever without per-site test files. Wire as part of `cargo test` so the existing CI test job picks it up. Add a CI step name making failures obvious.
5. **Doc fixes**: README drop the Shopping cell; `.env.example` refresh example model ids (keep it generic, don't hardcode pricing).
