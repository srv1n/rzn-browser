# EUI-T-0001 · Supervisor run store and run-history RPCs

## Intent

One local source of truth for every workflow run on this device — local CLI, MCP, and fleet-dispatched — persisted by the supervisor and queryable over RPC. This is the foundation for the dashboard Runs tab (EUI-T-0005), workflow health flags (EUI-T-0002), and the popup's recent-runs strip (EUI-T-0004).

## Context

- The runner is extracted at `crates/rzn_browser/src/workflow_runner/` — `execute_workflow(transport, sink, workflow, opts) -> RunResultV2`, with a 6-method default-no-op `RunEventSink`. The fleet loop (`supervisor_fleet.rs`) already persists `fleet_results/<job_id>.json` + a journal; CLI runs go through `native_runner.rs`. Today these histories are disjoint — unify them.
- Supervisor RPC precedent: `supervisor.rs` has a `dispatch` fn with match arms like `"fleet.status"` / `"fleet.disable"` delegating to `supervisor_fleet` RPC fns. Add new arms the same way.
- Storage root: `default_app_base_dir()` (same base the fleet journal/cache use).
- `RunResultV2` lives in `rzn_contracts::v2`.

## Contract

New module `crates/rzn_browser/src/run_store.rs`:
- `RunStore` with `append(record)`, `list(filter)`, `get(run_id)`, `gc(policy)`.
- `RunRecord` (index entry): `run_id` (uuid), `workflow_id`, `workflow_hash?`, `origin` (`local_cli` | `mcp` | `fleet:<job_id>`), `started_at`/`ended_at` (epoch ms), `status` (`succeeded|failed|cancelled|timed_out`), `failing_step_index?`, `error_class?`, `error_message` (bounded ≤2KB), `step_count`, `params_digest` (sha256 of params JSON), `result_ref` (path).
- Layout: `<base>/runs/index.jsonl` (append-only summaries) + `<base>/runs/<run_id>.json` (full `RunResultV2`) + `<base>/runs/<run_id>.params.json` (raw params, mode 0600 — exists ONLY to power "re-run with same params"; never included in diagnostics export, GC'd with the run).
- Retention GC: keep newest N runs (default 500) or max age 30 days, whichever prunes more; run on supervisor start and daily. Limits must read from the settings store once EUI-T-0008 lands (constant until then).
- Wire-in: the fleet loop writes a record at terminal result post; the CLI runner writes through the same store at run end. Implementer chooses `RunEventSink` impl vs a direct call at completion, but both paths MUST land in the same index with correct `origin`.
- RPC arms in `supervisor.rs` dispatch: `runs.list {limit≤200, offset?, workflow_id?, status?, origin?} -> {total, runs:[RunRecord]}` and `runs.get {run_id} -> {record, result: RunResultV2}`.
- Store behind a Mutex; index appends need not fsync (journal already covers fleet crash-safety); tolerate a truncated final line on load.

## File ownership

- OWN: `crates/rzn_browser/src/run_store.rs` (new), new RPC match arms in `supervisor.rs`, wiring lines in `supervisor_fleet.rs` and `native_runner.rs`.
- OFF-LIMITS: `extension/**`, `workflow_runner/` internals beyond sink hookup, `cloud.rs`.

## Constraints

- Raw params never appear in `index.jsonl`, logs, or any export — digest only; the 0600 params file is the sole copy.
- No new heavy dependencies: JSONL + serde (do not add rusqlite).
- No AI attribution anywhere.

## Dependencies

- None (first task of the epic).
