# EUI-T-0003 · Supervisor control, status, logs, and diagnostics RPCs

## Intent

Everything the popup and dashboard need beyond run history, as supervisor RPCs: a master pause switch, one status-snapshot call, run start/replay/cancel, log tail, redacted diagnostics export, and enrollment over RPC so the UI never requires a terminal. The extension UI (EUI-T-0004+) is a thin client over exactly these methods.

## Context

- RPC precedent: `supervisor.rs` `dispatch` match arms (`fleet.status`, `fleet.disable` → `supervisor_fleet` fns). The fleet loop already has cooperative cancellation (AtomicBool) and a claim path to gate on pause.
- `fleet_cli.rs` owns enrollment logic (HTTP to backend, writes `fleet_config.json` 0600) — extract a shared fn; do not duplicate the HTTP flow.
- Logging is `tracing`; feed the ring buffer with a custom Layer.
- Run store + health from EUI-T-0001/0002.

## Contract

RPC arms (all in `supervisor.rs` dispatch, implementations in a new `crates/rzn_browser/src/supervisor_control.rs` unless noted):
- `automation.pause {cancel_current?: bool}` / `automation.resume {}` — paused = fleet loop stops claiming AND new local runs are refused with a clear error; `cancel_current: true` also cooperatively cancels the in-flight run. Paused state persists across supervisor restart (small state file under `default_app_base_dir()`).
- `runs.cancel {run_id?}` — cooperative cancel of the current run.
- `runs.start {workflow_id, params?}` — start a local run via `execute_workflow` (workflow resolved from local catalog or cache); refused while paused; returns `{run_id}`.
- `runs.replay {run_id}` — re-run with the stored `<run_id>.params.json` from the run store; returns new `{run_id}`.
- `runs.get_failure_context {run_id} -> {console_tail?, screenshot_b64?, dom_excerpt?, capture_unavailable?}` — reads the `<run_id>.failure.json` written by EUI-T-0002.
- `status.snapshot {} -> {supervisor_version, native_host_connected, extension_connected, paused, now_running: {run_id, workflow_id, origin, step_index, step_total, elapsed_ms} | null, fleet: {enrolled, device_id?, status?, server_url?, last_poll_at?, last_poll_error?} | null, recent_runs: [≤5 summaries], flagged_workflows: count}` — ONE call renders the whole popup; keep it cheap (no disk scans per call — cache/derive from in-memory state plus run-store tail).
- `logs.tail {limit≤500, level?, component?, run_id?}` — in-memory ring buffer (~2000 entries) fed by a `tracing` Layer; entries `{ts, level, component, run_id?, message}`. New module `log_buffer.rs`.
- `diagnostics.export {} -> {path}` — zip under `<base>/diagnostics/<ts>.zip`: recent index.jsonl slice, last N run result JSONs, ring-buffer dump, versions, fleet config WITH `device_token` REDACTED. `*.params.json` files are NEVER included.
- `fleet.enroll {server_url, code}` — shared enroll fn (refactored out of `fleet_cli.rs`), returns device summary; `fleet.unenroll {}` — best-effort disable call then deletes `fleet_config.json`.

## File ownership

- OWN: `crates/rzn_browser/src/supervisor_control.rs` (new), `crates/rzn_browser/src/log_buffer.rs` (new), new dispatch arms in `supervisor.rs`.
- SHARED (small, additive): `fleet_cli.rs` (extract shared enroll fn), `supervisor_fleet.rs` (claim path checks paused flag), run-store read helpers.
- OFF-LIMITS: `extension/**`, `workflow_runner/` step semantics, `cloud.rs`.

## Constraints

- SECURITY: `device_token` and raw params must never appear in logs, snapshots, or the diagnostics zip (test-enforced).
- Snapshot must not block on network (fleet fields come from cached poll state).
- No AI attribution anywhere.

## Dependencies

- EUI-T-0001 (run store); EUI-T-0002 (failure context, flagged count — snapshot may stub `flagged_workflows: 0` until it lands).
