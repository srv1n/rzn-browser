# MBR-T-0032 · MB-033 add supervisor multi-bridge routing tests

## Intent

Add focused supervisor tests proving multiple native-host bridges coexist, route by explicit target/session binding, isolate failures and health, and fence reconnect/stale response behavior.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Registering bridge A then bridge B does not replace A. | Focused supervisor test |
| A2 | Explicit call to bridge A is received by A and not B. | Focused supervisor test |
| A3 | Command with no target and two bridges returns `AMBIGUOUS_BROWSER_TARGET`. | Existing/focused supervisor test |
| A4 | Session opened on Edge continues routing to Edge while Chrome is connected. | Existing/focused supervisor test |
| A5 | Stale response from old bridge epoch is dropped. | Existing supervisor test |
| A6 | Timeout/restart of bridge A does not drain bridge B pending calls and health remains isolated. | Existing supervisor test |
| A7 | Same browser instance reconnect can resume a bound session while a different instance cannot. | Focused supervisor test |

## Non-goals

- Live browser/native-host integration smoke tests; MB-034 covers that.
