# SEC-T-0004 · Fix tool_llm client: gpt-5 temperature, HTTP status check, timeout, log location

## Intent

The tool-calling LLM client in `rzn_plan/src/tool_llm/llm_client.rs` very likely 400s on every call (gpt-5 temperature), hides HTTP errors behind a misleading message, can hang forever (no timeout), and writes full prompts+responses to world-readable `/tmp`. Fix all four in this one file.

## Context (read before coding)

Audit anchors (2026-06-11, re-verify with grep):

1. `tool_llm/llm_client.rs:57-60, 90-117` — hardcodes model `gpt-5-mini-2025-08-07` with `temperature: 0.0`. The repo's own `rzn_plan/src/openai_client.rs:64-69` documents that the gpt-5 family rejects non-default temperature → the API returns 400.
2. Same file — `response.status()` is never checked; the JSON-parse path falls through to a generic "No tool calls in response" error, masking the 400 above. This combination means the client probably never worked against gpt-5 and the failure was misdiagnosed.
3. `tool_llm/llm_client.rs:39` (approx) — `reqwest::Client::new()` with no timeout; the autonomous loop awaiting this call can hang indefinitely.
4. `tool_llm/llm_client.rs:151-157` — appends full request+response (system prompt, page data) to `/tmp/llm_raw_{correlation_id}.jsonl`: predictable path, world-readable, never cleaned, and `correlation_id` is interpolated into the path.

Also note `rzn_plan/src/llm_autonomous.rs:910-916` reads `OPENAI_API_KEY`/`LLM_PROVIDER` from env directly, bypassing `PlanConfig` — align key sourcing with the rest of the crate while you are here (single config path).

## Implementation plan

1. Omit `temperature` from the request when the model id starts with `gpt-5` (mirror however `openai_client.rs` handles it — reuse its helper if one exists rather than duplicating the check).
2. After `send()`, check `response.status()`; on non-2xx read the body (bounded to ~2KB) and return `Err` containing status + body snippet. Only then attempt JSON parsing.
3. Build the `reqwest::Client` once (lazy static / OnceLock or struct field) with `.timeout(Duration::from_secs(120))` and `.connect_timeout(Duration::from_secs(10))`.
4. Raw-log: gate behind an env flag (e.g. `RZN_LLM_DEBUG_LOG=1`), write under the app data dir (see `rzn_core::runtime_paths` for the app base convention) instead of `/tmp`, create with mode 0600 on unix, and sanitize `correlation_id` to `[A-Za-z0-9_-]` before using it in a filename. If SEC-T-0005 has already landed its `write_secret_file`-style helper in rzn_core, use it.
5. Unit tests with a mock server (the crate already depends on reqwest; use `httptest`/`wiremock` if present in the tree, otherwise a tiny hyper stub): non-2xx surfaces status+body; gpt-5 request body contains no `temperature` key.
