# SEC-T-0002 · Escape workflow param substitution and migrate quote-interpolating workflows to args

## Intent

Workflow params are substituted into embedded JavaScript with zero escaping, so any value containing `'`, `\`, or a newline breaks the step, and a crafted value executes arbitrary MAIN-world JS. Make the substitution layer injection-safe and migrate every quote-interpolating workflow to the safe `args` channel.

## Context (read before coding)

Verified 2026-06-11 (re-check anchors before editing):

- `crates/rzn_browser/src/native_runner.rs:1033` — `substitute_string` is a raw multi-pass `out.replace(&format!("{{{}}}", key), val)`. No escaping, no context awareness. A duplicate lives at `crates/rzn_browser/src/cloud.rs` around line 1944.
- 13 workflows interpolate `'{param}'` directly inside `script` bodies. Known sites (audit line anchors): `workflows/instagram/instagram-dm-send.json:106` (`cleanArg('{message_text}')`), `instagram-post-comment.json:118`, `instagram-follow-account.json:141,167`, `instagram-search.json:116`, `instagram-profile-recent-posts.json:148`, `instagram-post-like.json:120`, `instagram-post-extract.json:120`, `workflows/google/google-search.json:75`, `google/google-lens.json:113`, `workflows/x/x_open.json:162,215`, `x/x_search_posts.json:102`, `workflows/bing/bing_images_search.json:150`, `workflows/apple_ads/portal-report.json:89`. Re-discover the full set with: `grep -rln "'{" workflows --include='*.json'` and inspect each hit inside `script` fields.
- The `cleanArg` helper inside workflow scripts only strips *unsubstituted* `{placeholder}` tokens — it is not an escaping mechanism.
- The safe channel already exists and is used correctly by `workflows/chatgpt/chatgpt_send.json`: pass values via the step's `args` array; they arrive as `arg0..arg2` in the eval wrapper. **Constraint:** the engine's eval wrappers declare only `arg0..arg2` — `arg3+` silently becomes `null`. For more than 3 params, pass one object arg or use `window.__rzn_params`.

Impact is dual: reliability (a DM body containing `don't` syntax-errors the Instagram step today) and security (params can originate from LLM-planned or remote-cloud runs, so this is a JS-injection surface, not just a quoting bug).

## Implementation plan

1. **Engine guard (root fix)**: in `substitute_string` callers, make substitution context-aware. For step fields that hold JavaScript source (`script` / eval bodies — confirm exact field names in the step schema, see `schema/actions-v1.json` and `crates/rzn_core/build.rs` codegen), refuse raw `{param}` substitution: return a structured workflow-validation error telling the author to use `args`. Keep plain substitution for non-code fields (URLs, selectors, text values).
2. **Escape helper**: add `substitute_into_js_string_literal` (JSON-escapes the value via `serde_json::to_string` and strips the surrounding quotes) for any legacy path that must keep interpolating; use it as the fallback if a hard refusal is too breaking — but prefer refusal + migration.
3. **Deduplicate**: `native_runner.rs` and `cloud.rs` must share one substitution implementation (move to a common module; the cloud duplicate has the same bug).
4. **Migrate the 13 workflows**: replace `'{param}'`-in-script with `args: ["{param}"]` + `arg0` usage (or `window.__rzn_params` for >3 values), mirroring `chatgpt_send.json`. Keep behavior identical otherwise; do not redesign the flows.
5. **Adversarial tests**: unit tests in `rzn_browser` for values containing `'`, `"`, `\`, newline, `</script>`, `{nested}` — both the refusal path and the escape helper. Extend the existing `*_workflows_parse_test.rs` pattern so all migrated workflows still parse and validate.
6. **Do not live-test against Instagram/X/Google by default** — these sites have aggressive bot filters. Prove via unit tests and workflow validation; a single live smoke (e.g. `rzn-browser run google search --param search_query="rust's borrow checker"`) only if the human asks.
