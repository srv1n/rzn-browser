# MBR-T-0027 · MB-028 add bridge lifecycle logging and metrics

## Intent

Add structured bridge lifecycle logs and status counters for registration, unregistration, target resolution, stale/mismatched responses, and reconnect decisions without logging sensitive payloads.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Bridge register/unregister logs identify bridges without tokens. | Focused observability test/source review |
| A2 | Ambiguous target errors are logged with candidate count and command name. | Focused counter/log path test |
| A3 | Stale response drops increment per-bridge counters visible through bridge status. | Focused counter test |
| A4 | Reconnect rebind decisions are logged with reason and outcome. | Focused source/counter test |
| A5 | Logs avoid full DOM snapshots, workflow payloads, tokens, and sensitive page content. | Focused source review |
| A6 | Tests verify counter increments where practical. | Automated test evidence |
