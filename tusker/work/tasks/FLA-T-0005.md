# FLA-T-0005 · CLI fleet commands: enroll, status, disable

## Intent

Add the operator-facing CLI surface for fleet enrollment and status on the laptop: `rzn-browser fleet enroll|status|disable`. This writes the `fleet_config.json` the supervisor's poll loop (FLA-T-0003) reads, completing the "type a code from the dashboard, laptop joins the fleet" onboarding flow.

## Context

- CLI is clap-based in `crates/rzn_browser/src/main.rs` (11k lines; subcommand enums near the top). Precedent for a config-writing cloud command set: `rzn-browser cloud pair` (`cloud.rs:513-557`) writing `LocalCloudActorConfig` (`cloud.rs:258-271`, version-tagged, via secure file helpers). Follow the same persistence pattern for the new config.
- Enrollment endpoint: `POST {server}/v1/fleet/enroll` with `rzn_contracts::fleet_v1::FleetEnrollRequestV1` → `FleetEnrollResponseV1` (token returned exactly once). Self view: `GET /v1/fleet/device/self` with the device token.
- Capabilities to report at enroll: reuse the supervisor capability vocabulary (e.g. flags listed around `supervisor.rs:64-71`) plus `fleet_poll_v1`; versions from the crate version and, when available, the extension version the supervisor last saw.
- Config file: `fleet_config.json` in the runtime dir — shape (shared contract with FLA-T-0003): `{ "version": "rzn.fleet.device_config.v1", "server_url", "device_id", "device_token", "tenant_id", "poll_interval_seconds"? }`, file mode 0600.

## Contract

New `fleet` subcommand group (`crates/rzn_browser/src/fleet_cli.rs`, thin dispatch from main.rs):

- `rzn-browser fleet enroll --server <url> --code <code> [--name <device-name>]`
  - Validates server URL with the same loopback-vs-https rule used by the cloud config (`cloudTransport`/`supervisor_cloud.rs:756-769` semantics: https required except loopback).
  - Gathers platform (`std::env::consts::OS` + arch), hostname default for `--name`, capabilities, versions; POSTs enroll; on success writes `fleet_config.json` (0600, atomic) and prints device_id + a "supervisor will start polling within a minute; restart it if it is already running" note. The token is never printed.
  - Errors (invalid/expired code, network) print a single actionable line and exit non-zero.
- `rzn-browser fleet status [--json]`
  - Reads config (absent → "not enrolled" exit 0); calls `GET /v1/fleet/device/self`; if the supervisor is running, also queries the local `fleet.status` RPC (FLA-T-0003) for loop state and journal tail (last 5 entries: job id + state). Prints device id, name, server, device status, last_seen, loop state. `--json` emits one machine-readable object.
  - Server unreachable → still print local config + loop state with a warning; exit 0.
- `rzn-browser fleet disable [--yes]`
  - Local opt-out: confirms (unless `--yes`), deletes `fleet_config.json`, tells a running supervisor to stop the loop via a `fleet.disable` local RPC if reachable (best effort), prints a reminder that server-side revocation is done by an operator in the dashboard/API.

Tests: config round-trip (write→read, 0600 mode asserted), enroll happy path + error path against a mock HTTP server, status rendering from a fixture config + mocked self response (assert `--json` shape), disable removes the file. Use the same mock-server approach as FLA-T-0003.

## File ownership

- OWN: `crates/rzn_browser/src/fleet_cli.rs` (new), the subcommand enum variants + dispatch arm in `main.rs` (small, clearly-scoped addition), `fleet.disable` RPC registration if FLA-T-0003 has not already provided it (coordinate — if both tasks add it, keep FLA-T-0003's).
- OFF-LIMITS: `supervisor_fleet.rs` internals, `cloud.rs` (reference only), `workflow_runner/**`, extension/**.

## Constraints

- Never print or log the device token; config file 0600; overwriting an existing enrollment requires `--force` (prevents accidental identity clobber of a working device).
- Keep main.rs churn minimal — logic lives in fleet_cli.rs.
- No AI attribution anywhere.

## Dependencies

- FLA-T-0001 (contracts). FLA-T-0003's local RPCs are consumed best-effort (feature-detect: method-not-found → "loop state unavailable").
