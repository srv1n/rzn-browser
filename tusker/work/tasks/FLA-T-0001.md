# FLA-T-0001 · Fleet wire contracts in rzn_contracts

## Intent

Add a versioned `fleet_v1` module to `rzn_contracts` defining every wire type exchanged between the backend fleet control plane and the laptop supervisor. This is the cross-repo contract: the backend consumes this crate via a sibling path dependency, so these shapes must be complete and stable before both sides build against them.

## Context

- Existing conventions to follow: `crates/rzn_contracts/src/v1.rs` (cloud command envelope types — serde snake_case, versioned struct names ending in `V1`, `#[serde(default)]` for forward-compat fields) and `v2.rs` (`WorkflowManifestV2`, `RunResultV2` at `v2.rs:889-908`).
- Design: pull model. Device polls with a health snapshot; response carries job assignments, cancellations, and the device's own status. Results post back a full `RunResultV2`. Enrollment redeems a one-time code for a `fld_`-prefixed device token.
- The backend (FLT-T-0003/0005/0006 in the backend repo's vault) codes against exactly the field names in this contract — coordinate by matching this file, not by importing backend code.

## Contract

New file `crates/rzn_contracts/src/fleet_v1.rs`, exported from `lib.rs` as `pub mod fleet_v1`:

```rust
pub const FLEET_PROTOCOL_VERSION: &str = "rzn.fleet.v1";

// Enrollment
pub struct FleetEnrollRequestV1 { pub code: String, pub device_name: String, pub platform: String,
    pub capabilities: Vec<String>, pub cli_version: String, pub extension_version: String }
pub struct FleetEnrollResponseV1 { pub device_id: String, pub device_token: String, pub tenant_id: String,
    pub poll_interval_seconds: u64, pub server_time_ms: i64 }

// Health (reuses supervisor readiness vocabulary)
pub struct DeviceHealthV1 { pub browser_running: bool, pub extension_bridge_up: bool,
    pub readiness_cause: Option<String>, pub cli_version: String, pub extension_version: Option<String>,
    pub uptime_seconds: u64, pub running_job_ids: Vec<String> }

// Poll
pub struct FleetPollRequestV1 { pub health: DeviceHealthV1, pub active_job_ids: Vec<String>, pub max_jobs: u32 }
pub struct FleetJobAssignmentV1 { pub job_id: String, pub workflow_id: String, pub workflow_hash: String,
    pub params: serde_json::Value, pub priority: i32, pub single_delivery: bool,
    pub execution_deadline_seconds: u64, pub lease_expires_at_ms: i64 }
pub struct FleetPollResponseV1 { pub jobs: Vec<FleetJobAssignmentV1>, pub cancellations: Vec<String>,
    pub poll_interval_seconds: u64, pub device_status: FleetDeviceStatusV1 }
pub enum FleetDeviceStatusV1 { Active, Dormant, Revoked }   // serde lowercase

// Workflow fetch
pub struct FleetWorkflowFetchResponseV1 { pub workflow_id: String, pub content_hash: String,
    pub manifest: serde_json::Value }

// Result post
pub struct FleetResultPostV1 { pub job_id: String, pub status: FleetJobTerminalStatusV1,
    pub run_result: crate::v2::RunResultV2, pub error: Option<String>,
    pub started_at_ms: i64, pub finished_at_ms: i64 }
pub enum FleetJobTerminalStatusV1 { Succeeded, Failed, TimedOut, Cancelled, Aborted } // serde snake_case
pub struct FleetResultAckV1 { pub ok: bool, pub job_status: String, pub deduped: bool }

// Device self view (CLI `fleet status`)
pub struct FleetDeviceSelfV1 { pub device_id: String, pub name: String, pub status: FleetDeviceStatusV1,
    pub tenant_id: String, pub last_seen_at_ms: Option<i64> }

// Error envelope shared by all fleet endpoints
pub struct FleetErrorV1 { pub code: String, pub message: String }
```

All types: `#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]`, `#[serde(rename_all = "snake_case")]` where enums need it, `#[serde(default)]` on every `Vec`/`Option`/numeric field that can safely default so old peers tolerate new fields. Document each struct with a one-line doc comment naming the endpoint that carries it. Known error `code` values (`device_revoked`, `device_dormant`, `job_not_claimed_by_device`, `enrollment_code_invalid`) listed as consts: `pub mod error_codes { pub const DEVICE_REVOKED: &str = "device_revoked"; ... }`.

## File ownership

- OWN: `crates/rzn_contracts/src/fleet_v1.rs` (new), one `pub mod fleet_v1;` line in `crates/rzn_contracts/src/lib.rs`.
- OFF-LIMITS: `v1.rs`, `v2.rs` (read-only reference), everything else in the repo.

## Constraints

- Serde round-trip unit tests for every struct (serialize → deserialize → PartialEq) plus one JSON snapshot literal per top-level type pinning the exact wire field names (so accidental renames fail the test).
- No behavior code in the contracts crate — types, consts, tests only.
- No AI attribution anywhere.
