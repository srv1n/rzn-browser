# MBR-T-0015 · MB-016 implement target resolution for browser commands

## Intent

Resolve browser command targets deterministically before dispatching to the native-host bridge. Browser commands must route by explicit bridge ID, browser instance ID, browser kind, session binding, or a safe singleton fallback, and must return structured target errors when selection is missing or ambiguous.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | `bridge_id` / `supervisor_bridge_id` selects that connected bridge or returns `BRIDGE_NOT_FOUND`. | Focused supervisor test |
| A2 | `browser_instance_id` selects the active bridge for that instance or returns `BROWSER_INSTANCE_NOT_CONNECTED`. | Focused supervisor test |
| A3 | `browser` / browser kind selects exactly one matching connected bridge. | Focused supervisor test |
| A4 | Browser kind with multiple matching bridges returns `AMBIGUOUS_BROWSER_TARGET` including candidate bridge IDs. | Focused supervisor test |
| A5 | No explicit target preserves singleton fallback and rejects multiple connected bridges with `AMBIGUOUS_BROWSER_TARGET`. | Focused supervisor test |
| A6 | Browser command dispatch forwards a structured resolved target object for logging/downstream calls. | Focused supervisor test |

## Non-goals

- Binding sessions across reconnects. That starts in MB-017/MB-019.
- Public CLI browser target flags. That is MB-024.
