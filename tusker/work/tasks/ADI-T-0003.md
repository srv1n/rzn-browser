# ADI-T-0003 · Google Ads Transparency Center workflow pack

## Intent

Workflow pack over `adstransparency.google.com` — the Google-side complement to the Meta Ad Library pack. It is a public surface with no API at all, so a browser pack is the only route. Search an advertiser, enumerate their ads with format, date information, and regions, and emit the same `ads-manifest.json` contract defined in ADI-T-0001 with `source: "google_ads_transparency"`.

Coverage differs from Meta: Google surfaces format (text/image/video), shown-date ranges, and regions, but less ad copy. Map fields best-effort and leave nulls; the shared schema already treats everything except `id` and `source` as nullable.

## Scope

In:
- Advertiser search (typeahead resolution to a verified advertiser entity) with region parameter.
- Enumeration of the advertiser's ads: format, first/last-shown dates where surfaced, regions, creative preview URLs where present; pagination to a configurable cap.
- Manifest emission against the shared schema fixture; `started_at` mapped from first-shown date.
- Fixture-based contract test per the WCP harness.

Out:
- Creative asset downloading beyond recording preview `source_url`s (extend ADI-T-0002's approach later if previews prove downloadable and valuable).
- Political ad transparency reports and region-specific legal disclosures.

## Constraints

- Content-script-first, CDP only as fallback, human-scale pacing — same doctrine as ADI-T-0001.
- Logged-out access assumed; no Google session dependency.
- Selectors/payload parsing centralized for one-file drift repair.
