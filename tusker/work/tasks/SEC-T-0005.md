# SEC-T-0005 · Write all secret files 0600 and move LLM/oversize debug artifacts out of world-readable tmp

## Intent

Several credential and sensitive-data files are written with default permissions (0644, world-readable on multi-user unix) or into world-readable `/tmp`. Introduce one shared secure-write helper in `rzn_core` and use it at every secret-writing site; move debug artifacts out of `/tmp` and add cleanup.

## Context (read before coding)

Audit anchors (2026-06-11, re-verify each with grep before editing):

1. `crates/rzn_native_host/src/cloud.rs:619-630` — `persist_local_actor_config()` writes `cloud_actor_v1.json` containing the `actor_token` bearer credential via `std::fs::write` (0644).
2. `crates/rzn_browser/src/supervisor.rs:4615-4623` — `get_or_create_token` writes the local-socket auth token 0644.
3. `crates/rzn_plugin_devkit/src/main.rs:223` — writes the **ed25519 private signing key** 0644.
4. `crates/rzn_native_host/src/main.rs:662-694` — `write_oversize_response_artifact()` dumps full extension responses (may contain cookies/DOM/credentials) to `std::env::temp_dir()/rzn-native-host-artifacts/<uuid>.json`, default perms, never deleted. `/tmp` is world-readable on Linux.
5. `crates/rzn_plan/src/llm.rs:1086-1106` (also ~1176, ~1233) — appends full raw LLM responses (which embed page data) to `~/rzn_build.log`, unbounded, no rotation, misleading name.

Note: `/tmp/llm_raw_*.jsonl` in `tool_llm/llm_client.rs` is handled by SEC-T-0004 — do not double-edit that file; if this task lands first, SEC-T-0004 will consume the helper added here.

## Implementation plan

1. **Helper in rzn_core** (so all crates share it): `write_secret_file(path, bytes)` — creates parent dirs with mode 0700, writes via `OpenOptions::new().write(true).create(true).truncate(true).mode(0o600)` (`std::os::unix::fs::OpenOptionsExt`); on Windows fall back to plain write (default user-scoped ACLs are acceptable). Also `secure_artifact_dir()` returning a 0700 per-user dir under the app base (`rzn_core::runtime_paths` already resolves `<app_base>/run` and `<app_base>/secure` — follow that convention).
2. Replace sites 1-3 with the helper. For pre-existing token/key files, chmod them to 0600 on next write/read (best-effort, log on failure).
3. Site 4: write oversize artifacts under `secure_artifact_dir()` with 0600 and add TTL cleanup (delete files older than 24h on host startup; cap the dir at e.g. 50 files).
4. Site 5: gate the raw-response log behind an env flag, move it under the app data dir with 0600, cap file size (truncate or rotate at ~10MB), and rename to something honest like `llm_responses.debug.log`.
5. Unix unit tests: assert `metadata.permissions().mode() & 0o777 == 0o600` for files and `0o700` for dirs created by the helper; assert TTL cleanup removes old artifacts.
