# EUI-T-0005 · Dashboard Runs tab and run detail

## Intent

The dashboard Runs tab: unified execution history (local + fleet in one list with an origin column) and a run-detail view with step timeline, error context, result viewer, and one-click re-run. This is the debug loop for the whole system.

## Context

- Pure frontend task: all data comes from supervisor RPCs that already exist by now — `runs.list`, `runs.get`, `runs.replay`, `runs.get_failure_context`, `workflows.health` (for the workflow filter dropdown).
- Scaffold, routing, rpc client, and styles from EUI-T-0004.
- Run records carry `origin` (`local_cli` | `mcp` | `fleet:<job_id>`), status, failing_step_index, error_class, fingerprint.

## Contract

- `#runs` list: paginated 50/page via `runs.list`; columns: status dot, workflow, origin chip, started (relative), duration, failing step (when failed). Filters: workflow (dropdown), status, origin; client-side text filter on the current page.
- `#runs/<run_id>` detail:
  - Step timeline from `RunResultV2` steps: per-step status + duration, failing step highlighted.
  - Error block: error_class chip, message, fingerprint, "first seen" if the same fingerprint appears in listed history.
  - Failure context via `runs.get_failure_context`: console tail (monospace, collapsed by default), screenshot (if present), DOM excerpt; render `capture_unavailable` reason otherwise.
  - Result JSON viewer: collapsible tree or pretty-printed block with copy button.
  - "Re-run with same params" → `runs.replay` → navigate to the new run's detail; disabled with tooltip when the supervisor is paused or the workflow is no longer available.
- Empty/error states: no runs yet (call to action to run a workflow), `SupervisorUnreachable` skeleton.

## File ownership

- OWN: `extension/src/dashboard/runs/**` (new).
- SHARED (additive): `extension/src/ui/rpc.ts` zod schemas for the methods used.
- OFF-LIMITS: `crates/**`, `background.ts`, other dashboard tabs.

## Constraints

- No new dependencies; vanilla TS rendering consistent with the scaffold.
- Screenshots render from base64 payloads — never fetch remote URLs.
- No AI attribution anywhere.

## Dependencies

- EUI-T-0004 (scaffold), EUI-T-0001/0002/0003 (RPCs).

- No live streaming of an in-flight run's steps (popup covers "now running"; refresh covers the rest), no run deletion UI, no artifact preview beyond what failure context carries.
