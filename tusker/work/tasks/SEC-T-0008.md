# SEC-T-0008 · Refactor background broker dispatch to command registry with allowlist and sender validation

## Intent

`background.ts` executes whatever command the native host sends through a ~2000-line if/else with no allowlist, and its `chrome.runtime.onMessage` handler performs zero sender validation on privileged operations. Refactor dispatch into a command-registry table with explicit per-source allowlists and sender checks, fix the hanging `native_input` path, redact payload logging, and strip prod test hooks.

## Context (read before coding)

All anchors in `extension/src/background.ts` (7744 lines; audit 2026-06-11 — re-verify with grep):

1. `:3268-5445` — `handleBrokerMessage` giant if/else. Executes any native-host cmd: `eval_with_scripting`/`eval_with_cdp` (arbitrary MAIN-world JS in any tab), `enable_debug` (CDP attach), downloads, navigation, cookie/storage writes. The native host bridges to a remote cloud actor after popup pairing, so this is transitively remote-reachable. No unknown-command rejection, no per-command policy.
2. `:7181-7596` — `chrome.runtime.onMessage`: no `sender` checks on `cloud_ui_issue_pairing_code` / `cloud_ui_pair_actor` / `cloud_ui_run_remote_command` / `cloud_ui_disconnect`, `eval_with_scripting`, `eval_with_cdp`, `download_image`, `native_input`. Any content script (injected into `<all_urls>`) can call all of them. (`externally_connectable` is unset, so raw pages can't — the content-script path is the exposure.)
3. `:7443` — `else if (message.cmd === 'native_input' && nativePort)`: when `nativePort` is null no branch matches → no `sendResponse` → caller hangs/errors opaquely.
4. Logging leaks: `:3560` logs full broker message (500 chars), `:5872` full task structure, `:3409` full broker payloads; these carry eval scripts, page content, possibly credentials, and also flow into the unified native log.
5. `:5745-5758` — `globalThis.sendTestPing`, `reconnectNative`, `__rznTestRunWorkflowSteps` ship in production builds.
6. `:7320` — `take_screenshot` falls back to the active tab when sender has no tab → can capture an unrelated foreground tab.

Structure map for orientation (from the audit): CDP lease mgr `:31-292`; native transport `:350-665, 3356-3489, 5603-5675`; broker leases/watchdog `:689-1264`; eval engine `:2489-3060`; `handleBrokerMessage` `:3268-5445`; `executeWorkflow` `:5855-7004`; cloud UI `:7012-7178`; onMessage `:7181-7596`.

## Implementation plan

Work in mechanical stages; keep each stage green (`bun x vitest run` + build) before the next:

1. **Registry extraction**: create `src/commands/registry.ts` with `type CommandSource = 'native' | 'popup' | 'content'` and `interface CommandDef { handler(msg, ctx): Promise<Resp>; allowedSources: CommandSource[]; }`. Move `handleBrokerMessage` branches into table entries one group at a time (eval, tabs/nav, downloads, cookies/storage, debug/CDP, cloud, misc). Pure code motion first — no behavior change, diff reviewable.
2. **Allowlist enforcement**: dispatch looks up the table; unknown cmd → structured error response `{ok:false, error:'unknown_command'}` (logged at warn with cmd name only). Native-host source may call the broker command set; popup source only `cloud_ui_*` + status; content source only the narrow set it genuinely uses today (enumerate by grepping `chrome.runtime.sendMessage` in `contentScript.ts` and `src/content/`).
3. **Sender classification**: in `onMessage`, derive source: popup ⇒ `sender.id === chrome.runtime.id && !sender.tab`; content ⇒ `sender.id === chrome.runtime.id && sender.tab != null`; reject otherwise. `cloud_ui_*` requires popup. Eval/download/native_input from content must carry the expected shape and use `sender.tab.id` as the target tab (no caller-chosen tabId from content).
4. **native_input fix**: always respond; if `!nativePort` ⇒ `sendResponse({ok:false, error:'native host disconnected'})`.
5. **take_screenshot**: require explicit tabId from native source; for content source, use `sender.tab.id` only — remove active-tab fallback.
6. **Log redaction**: replace payload logs with `{cmd, req_id, size}`; add a `redactForLog()` helper for the few places needing more.
7. **Test hooks**: wrap `globalThis.*` test exports in the dev-build flag used by the build scripts (or introduce one via `build.sh` define, consistent with SEC-T-0001's flag).
8. **Tests**: vitest unit tests for the registry (unknown cmd, wrong source, popup-only enforcement, native_input disconnected). Then full e2e: the extension's Playwright suites exercise the broker path heavily — they are the regression gate for the code motion.
