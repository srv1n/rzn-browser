# MBR-T-0013 · MB-014 route native extension calls to selected bridge

## Intent

Route every supervisor native extension call through an explicit or safely resolved bridge target so multi-browser calls cannot silently jump between bridges.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Explicit bridge ID sends the request only to that bridge and includes selected bridge ID/epoch in JSON. | Focused supervisor test |
| A2 | Exactly one connected bridge with no explicit target preserves existing routing. | Focused supervisor test |
| A3 | Multiple connected bridges with no explicit target returns `AMBIGUOUS_BROWSER_TARGET`. | Focused supervisor test |
| A4 | Wrong bridge/stale epoch responses are ignored and counted against the targeted bridge health. | Existing focused supervisor test |
| A5 | Send failure clears/reconnects only the selected bridge. | Focused supervisor test |

## Non-goals

- Browser preference policy beyond bridge-id targeting and safe singleton fallback.
