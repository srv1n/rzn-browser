# EUI-T-0007 · Dashboard Fleet tab and enrollment UX

## Intent

The dashboard Fleet tab: enrollment without a terminal, device identity and liveness at a glance, and unenroll/reconnect controls. When not enrolled this tab IS the enrollment CTA — the rest of the dashboard works identically either way (progressive disclosure, no separate fleet mode).

## Context

- Supervisor RPCs from EUI-T-0003: `fleet.enroll {server_url, code}`, `fleet.unenroll {}`, plus existing `fleet.status` / `fleet.disable`; `status.snapshot` carries cached fleet state (status, last_poll_at, last_poll_error).
- Enrollment codes are minted operator-side (backend console); the device user only pastes a code. Dormant/revoked devices stop claiming; reconnecting a dormant device is operator reactivation OR re-enrollment with a fresh code — the UI must explain which applies.
- Fleet job counts derive from the run store (`runs.list` filtered `origin: fleet`).

## Contract

- Unenrolled state: short explainer of what enrolling does ("lets your team dispatch workflows to this browser"), fields for server URL + enrollment code → `fleet.enroll` → success screen showing resolved tenant/device name → enrolled state. Errors (bad code, unreachable server, expired code) render inline with the server's error message.
- Enrolled state:
  - Device card: device name/id, tenant, status chip (active / dormant / revoked), server URL, enrolled date, last successful poll age, last_poll_error (if any) in a warning strip.
  - Counters: fleet jobs run / succeeded / failed (run-store derived).
  - Master pause state surfaced with a link to the popup toggle (single master switch in v1 — no separate claim-only pause).
  - Dormant/revoked: banner explaining the state + the recovery path — "ask an operator to reactivate this device, or re-enroll with a new code" (re-enroll flows into the unenrolled form, keeping server URL prefilled).
  - Unenroll: confirm dialog ("stops cloud dispatch to this device and deletes its token") → `fleet.unenroll` → back to unenrolled state.
- Poll `status.snapshot` every 5s while the tab is visible.

## File ownership

- OWN: `extension/src/dashboard/fleet/**` (new).
- SHARED (additive): `extension/src/ui/rpc.ts` schemas.
- OFF-LIMITS: `crates/**` (all RPCs already exist), other dashboard tabs.

## Constraints

- The device token must never be displayed or logged by the UI — the RPC layer never returns it.
- No AI attribution anywhere.

## Dependencies
