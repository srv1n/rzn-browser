# MBR-T-0028 · MB-029 add Chrome Chromium Edge extension manifest overlays

## Intent

Add Chrome, Chromium, and Edge manifest overlays to the extension build so all three Chromium-family targets are generated from the shared base manifest while preserving per-target metadata.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Chrome, Edge, and Chromium manifests are generated from `manifest.base.json` plus a target overlay. | Manifest generator test |
| A2 | Required MV3 fields, permissions, background service worker, and native messaging permission survive generation for all three targets. | Manifest generator test |
| A3 | Each generated manifest reports the intended extension target through build metadata. | Manifest generator test |
| A4 | Shared permissions and content scripts remain sourced from `manifest.base.json`, not repeated in Chromium overlays. | Manifest source review/test |
| A5 | Chrome output has no Edge-specific release metadata. | Manifest generator test |

## Non-goals

- Firefox manifest migration or Firefox build behavior.
- Native-host install/registration support for Edge or Chromium.
