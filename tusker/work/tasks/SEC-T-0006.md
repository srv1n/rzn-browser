# SEC-T-0006 · Verify release artifact checksums in installers and scope quarantine-stripping

## Intent

The install pipeline is trust-on-first-use end to end: `curl | sh` fetches an unsigned tarball with no checksum verification, installers ad-hoc self-sign binaries, and quarantine attributes are stripped unconditionally. The release build already emits `.sha256` sidecars that nothing checks. Wire verification into the installers and guard the destructive paths.

## Context (read before coding)

Audit anchors (2026-06-11, re-verify):

1. `install.sh:51` (entry advertised at `README.md:166` as `curl -fsSL ... | sh`) downloads `releases/latest/download/*.tar.gz` and executes its bundled `install.sh` — no checksum, no signature.
2. `install.ps1:24` — same pattern via `irm ... | iex`.
3. `scripts/release/build_release_artifacts.py:155` — already generates `.sha256` sidecar files per artifact; they are uploaded but never consumed.
4. `scripts/release/install-runtime.sh:50` and `scripts/bundle/install-macos.sh:40` — `codesign --sign -` (ad-hoc) on installed binaries. Note MBR-T-0041 added signature *repair* here on purpose (macOS SIGKILL fix) — do not regress that; the repair stays, but it must operate on a checksum-verified artifact.
5. `setup.sh:76` and `install-runtime.sh:46` — `xattr` strip of `com.apple.quarantine`/`com.apple.provenance` on every installed binary, unconditionally.
6. `rm -rf` on env-derived paths with no guard: `install-runtime.sh:91` (`$EXT_DIR`), `install-macos.sh:65`, `setup.sh:227` — `RZN_RUNTIME_DIR`/`RZN_BUNDLE_INSTALL_ROOT` overridable; an empty var expands to `rm -rf /...` shapes.

## Implementation plan

1. **Checksum verification in `install.sh`**: after downloading the tarball, download its `.sha256` sidecar from the same release and verify (`shasum -a 256 -c` on macOS, `sha256sum -c` on Linux — detect which exists). Abort with a clear message on mismatch or missing sidecar. Same logic in `install.ps1` via `Get-FileHash`.
2. **Pin-by-default**: support `RZN_INSTALL_VERSION=vX.Y.Z`; when unset, resolve `latest` but print the resolved tag + checksum so installs are auditable.
3. **Quarantine scoping**: only strip quarantine when the artifact came from the verified-download path (pass an explicit flag from the caller); skip the strip for arbitrary pre-existing binaries. Keep the local-dev path in `setup.sh` working for locally built binaries (those have no quarantine attr anyway — guard with a comment).
4. **rm -rf guards**: before every `rm -rf "$VAR"`, validate: non-empty, absolute, depth ≥ 3 path components, and under `$HOME` or the expected install root. Add a small `safe_rm_dir()` shared function (these scripts already share helpers? check `scripts/` for a common lib; if none, duplicate the 6-line guard per script — POSIX sh compatible, `install.sh` runs under `sh`).
5. **Tests**: a local fixture test script (`test/manual` or a new `scripts/release/test_install_verification.sh`) that builds a fake release dir, serves it via `python3 -m http.server` or `file://`, tampers one byte, and asserts the installer refuses. Plus `bash -n` / `sh -n` on every touched script and `shellcheck` if installed.

Real Developer ID signing + notarization is intentionally out of scope (needs an Apple cert a human must provision) — if you finish everything else, add a Tusker gate proposing it rather than attempting it.
