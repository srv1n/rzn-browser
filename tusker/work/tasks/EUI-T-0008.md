# EUI-T-0008 · Dashboard Logs and Settings tabs

## Intent

The dashboard Logs and Settings tabs: a filterable live log viewer with one-click diagnostics export (the thing that makes remote fleet support survivable), and a small settings surface persisted supervisor-side.

## Context

- `logs.tail` and `diagnostics.export` RPCs from EUI-T-0003.
- Settings need two new supervisor RPC arms (owned here): `settings.get {}` / `settings.set {patch}` persisting to a settings file under `default_app_base_dir()`. Consumers: run-store GC (retention values from EUI-T-0001) and notifications (EUI-T-0009) read these settings.
- Settings v1 keys: `run_retention_count` (default 500), `run_retention_days` (default 30), `notifications_enabled` (default true), `notify_on` (`all` | `failures_only`, default `all`), `fleet_keep_window_on_failure` (default false; consumed by EUI-T-0009).

## Contract

- Supervisor side: `settings.get`/`settings.set` arms with validation (bounds: retention count 10..10000, days 1..365); unknown keys rejected; file read once at startup and cached; run-store GC re-reads effective values per GC pass.
- `#logs` tab:
  - Viewer over `logs.tail`: level filter, component filter, run-id filter (prefillable via deep link `#logs?run=<id>` from run detail), auto-refresh toggle (2s), pause-on-scroll, copy-line.
  - "Export diagnostics" button → `diagnostics.export` → renders the returned zip path with a copy button and a note that the archive is token- and params-redacted.
- `#settings` tab: retention fields, notifications toggle + notify_on radio, keep-window-on-failure toggle; save via `settings.set` with optimistic UI + error rollback; read-only display of poll interval and config path (env-controlled — shown for debugging, not editable).

## File ownership

- OWN: `extension/src/dashboard/logs/**` (new), `extension/src/dashboard/settings/**` (new), `settings.get`/`settings.set` supervisor arms + settings persistence module.
- SHARED (additive): `extension/src/ui/rpc.ts` schemas; run-store GC reads settings (coordinate with EUI-T-0001 constants).
- OFF-LIMITS: other dashboard tabs, `workflow_runner/`.

## Constraints

- Log viewer renders at most the returned window (≤500) — no unbounded DOM growth with auto-refresh on.
- Settings file must not contain secrets; it is safe to include in diagnostics export.
- No AI attribution anywhere.

## Dependencies

- EUI-T-0004 (scaffold), EUI-T-0003 (`logs.tail`, `diagnostics.export`).
