# MBR-T-0017 · MB-018 add clear ambiguous and missing target errors

## Intent

Return structured, actionable browser-target errors for missing, ambiguous, and conflicting multi-browser routing inputs, preserving machine-readable JSON while giving CLI users concrete next steps.

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Zero connected bridges returns `NO_BROWSER_BRIDGE_CONNECTED` without ambiguity wording. | Focused supervisor test |
| A2 | Multiple connected bridges with no target returns `AMBIGUOUS_BROWSER_TARGET`. | Focused supervisor test |
| A3 | Ambiguous errors include candidate bridge ID, browser instance ID, extension target hint, extension ID, and last health timestamp. | Focused supervisor test |
| A4 | Human-readable formatting suggests `--browser`, `--browser-instance`, and `--bridge` when candidates support them. | Focused formatter test |
| A5 | JSON result exposes the same candidate list and next steps without terminal formatting. | Focused supervisor test |
| A6 | Missing bridge, missing browser instance, and session target conflict use stable error codes. | Focused supervisor test |

## Non-goals

- Adding public CLI target flags. That is MB-024.
- End-to-end browser smoke tests. That is MB-034.
