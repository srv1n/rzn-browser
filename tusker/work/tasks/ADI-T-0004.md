# ADI-T-0004 · Ads packs smoke-test lane (selector drift detection)

## Intent

Browser packs are DOM contracts that Meta or Google can break on any deploy. Once clients pay for scheduled ad-intel runs, we must learn about selector drift before their Monday report silently comes back empty. This task adds a smoke lane for the ads packs: a cheap scheduled run per pack against a stable, high-volume advertiser that asserts a non-empty, schema-valid envelope and fails loudly with an actionable drift report.

## Dependencies

- ADI-T-0001 (Meta Ad Library pack) and ADI-T-0003 (Google Transparency Center pack) must exist; the lane covers whichever packs have landed.

## Scope

In:
- A smoke command per ads pack: run at a small cap (e.g. 5 ads) against a pinned stable advertiser, validate the envelope against the schema fixture, exit nonzero on empty or invalid output.
- Machine-readable drift report on failure: which fields went missing or null-rated beyond baseline, so the repair agent knows where to look.
- Documented cadence and invocation (make target or existing scheduler convention in this repo); wire into the WCP harness lanes as the live-site lane for these packs.

Out:
- Auto-repair of selectors.
- Alert delivery (email/Slack) — operators wire the nonzero exit into whatever alerting exists.
- General smoke coverage for non-ads packs (pattern can be lifted later).

## Acceptance

| ID | Outcome | Proof |
|---|---|---|
| A1 | Smoke command exits 0 on a healthy pack and nonzero on empty/schema-invalid output. | Focused test with a stubbed invalid envelope; one live healthy run capsule. |
| A2 | Failure output names the missing/degraded fields against the baseline. | Focused test asserting the drift report content. |
| A3 | Cadence and invocation documented; runnable in one command. | Docs + command check. |

## Non-goals

- No guarantee of catching drift that still yields schema-valid output with subtly wrong values; baselines are structural, not semantic.
