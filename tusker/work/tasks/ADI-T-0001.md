# ADI-T-0001 · Meta Ad Library workflow pack: search and enumerate ads

## Intent

Workflow pack over the public Meta Ad Library (`facebook.com/ads/library`). This is the only first-party route to commercial competitor ads: the official Ad Library API covers only political/issue ads and EU-targeted ads, so the web UI is the front door, not a workaround. The surface is publicly viewable without login.

The pack searches by advertiser page or keyword (with country filter) and enumerates active ads into a structured envelope plus an `ads-manifest.json` consumed by the rzn-tools ADS epic (creative analysis pipeline). Creative asset download is a separate task (ADI-T-0002); this task records asset `source_url`s where they are cheaply discoverable in the DOM, and may leave them null otherwise.

## Output contract (canonical definition — rzn-tools ADS-T-0001 mirrors it)

```json
{
  "source": "meta_ad_library",
  "captured_at": "ISO-8601",
  "query": { "advertiser": "...", "keyword": "...", "country": "US" },
  "ads": [{
    "id": "library ad id",
    "advertiser": "page name",
    "started_at": "YYYY-MM-DD or null",
    "active": true,
    "platforms": ["facebook", "instagram"],
    "media_type": "video | image | carousel | text",
    "copy": "primary text or null",
    "headline": "string or null",
    "cta": "string or null",
    "landing_url": "url or null",
    "assets": [{ "kind": "video|image", "path": null, "source_url": "url or null" }]
  }]
}
```

`started_at` matters most downstream: ad longevity is the primary "proven winner" signal in the analysis stage.

## Scope

In:
- Search by advertiser page (exact page selection from typeahead) and by keyword, each with a country parameter.
- Enumeration of active ads with the fields above; pagination/infinite-scroll to a configurable cap (default in the tens, not thousands — human-scale).
- Envelope + manifest emission per existing pack conventions; JSON schema fixture for the manifest.
- Fixture-based contract test per the WCP harness (no live site in tests).

Out:
- Asset downloading (ADI-T-0002).
- Political ads API, EU DSA data, login-gated surfaces.
- Any interaction with the user's Facebook session — this pack must assume logged-out access.

## Constraints

- Content-script-first execution; escalate to CDP only where extraction is otherwise infeasible, per repo doctrine.
- Human-scale pacing with jitter between page fetches; single-business research volume, not farming.
- Selectors and any GraphQL-payload parsing must be centralized in the pack so drift repairs are one-file fixes.

## Acceptance

|---|---|---|

| A2 | Keyword + country search works and records the query in the manifest. | Live proof run capsule. |
| A3 | Pagination honors the configurable cap and stops cleanly. | Live proof run at a small cap. |
| A4 | Manifest validates against the schema fixture. | Schema validation in the contract test. |
| A5 | Fixture-based contract test passes without any live site. | Test command output. |

## Non-goals

- No completeness guarantee across all of an advertiser's ads; the cap bounds the crawl.
- No historical/inactive ad recovery.
